\section{ওয়েব ডিজাইনের ধারণা}

\begin{learningbox}
এই টপিক শেষে শিক্ষার্থী পারবে:
\begin{itemize}
\item ওয়েব ডিজাইন কী তা সংজ্ঞাসহ ব্যাখ্যা করতে।
\item ওয়েবপেজ, ওয়েবসাইট, ওয়েব সার্ভার ও ওয়েব ব্রাউজারের ধারণা লিখতে।
\item ওয়েব ডিজাইনের উদ্দেশ্য, উপাদান, সুবিধা ও ব্যবহারক্ষেত্র ব্যাখ্যা করতে।
\item HTML, CSS ও JavaScript-এর ভূমিকা আলাদা করতে।
\item ভালো ওয়েব ডিজাইনের বৈশিষ্ট্য ও exam focus point মনে রাখতে।
\end{itemize}
\end{learningbox}

\subsection{ভূমিকা}

বর্তমান যুগে শিক্ষা, ব্যবসা, সংবাদ, ব্যাংকিং, চিকিৎসা, বিনোদন, যোগাযোগ এবং সরকারি সেবা - প্রায় সব ক্ষেত্রেই ওয়েবসাইট ব্যবহার করা হয়। কোনো ব্যক্তি, প্রতিষ্ঠান, স্কুল, কলেজ, সংবাদমাধ্যম বা ব্যবসা প্রতিষ্ঠান তাদের তথ্য internet-এর মাধ্যমে মানুষের কাছে পৌঁছে দিতে ওয়েবসাইট ব্যবহার করে।

ওয়েবসাইট তৈরি করার জন্য প্রথমে ওয়েবপেজ পরিকল্পনা ও ডিজাইন করতে হয়। একটি ওয়েবপেজে কোন তথ্য কোথায় থাকবে, কোন অংশে ছবি বা link থাকবে এবং user কীভাবে page ব্যবহার করবে - এসব আগে ঠিক করা হয়।

এরপর text, image, link, list, table ও form সাজিয়ে page-টি ব্যবহারবান্ধব করা হয়। ব্যবহারকারী যেন সহজে প্রয়োজনীয় তথ্য খুঁজে পায়, সেটিও ওয়েব ডিজাইনের গুরুত্বপূর্ণ অংশ।

HSC ICT Chapter 4-এ মূলত ওয়েব ডিজাইনের ধারণা, ওয়েবসাইটের কাঠামো, HTML-এর মৌলিক বিষয়, HTML tag ও syntax, text formatting, list, hyperlink, image, table, form, webpage designing এবং website publishing আলোচনা করা হয়।

\subsection{ওয়েব ডিজাইন কী?}

\begin{definitionbox}{ওয়েব ডিজাইন}
ওয়েবসাইট বা ওয়েবপেজের তথ্য, ছবি, লিংক, টেবিল, মেনু ও অন্যান্য উপাদানকে নির্দিষ্ট কাঠামো অনুযায়ী সুন্দর ও ব্যবহারবান্ধবভাবে সাজানোর প্রক্রিয়াকে ওয়েব ডিজাইন বলে।
\end{definitionbox}

আরও সহজভাবে বলা যায়, internet-এ প্রদর্শনের জন্য ওয়েবপেজ বা ওয়েবসাইট তৈরি ও সাজানোর কৌশলই হলো ওয়েব ডিজাইন। ওয়েব ডিজাইনে শুধু page সুন্দর দেখানোই উদ্দেশ্য নয়; বরং তথ্যগুলো যেন logical order-এ থাকে, navigation সহজ হয়, text পড়তে সুবিধা হয় এবং user দ্রুত প্রয়োজনীয় তথ্য পায় - সেটিও গুরুত্বপূর্ণ।

\begin{boardbox}{পরীক্ষার সংক্ষিপ্ত উত্তর}
ওয়েবসাইট বা ওয়েবপেজের তথ্য, ছবি, লিংক, টেবিল, মেনু ও অন্যান্য উপাদানকে নির্দিষ্ট কাঠামো অনুযায়ী সুন্দর ও ব্যবহারবান্ধবভাবে সাজানোর প্রক্রিয়াকে ওয়েব ডিজাইন বলে। এর মাধ্যমে internet-এ প্রদর্শনের উপযোগী web page তৈরি করা হয়।
\end{boardbox}

\subsection{ওয়েব ডিজাইনের মূল উদ্দেশ্য}

ওয়েব ডিজাইনের উদ্দেশ্য শুধু সুন্দর page তৈরি করা নয়; বরং তথ্যকে এমনভাবে সাজানো যাতে ব্যবহারকারী দ্রুত, সহজে এবং স্পষ্টভাবে প্রয়োজনীয় তথ্য পায়।

\begin{keypointbox}{প্রধান উদ্দেশ্য}
\begin{itemize}
\item তথ্য সুন্দর ও সংগঠিতভাবে উপস্থাপন করা।
\item ব্যবহারকারীর জন্য সহজ navigation তৈরি করা।
\item ওয়েবসাইটকে আকর্ষণীয় ও user-friendly করা।
\item প্রয়োজনীয় তথ্য দ্রুত খুঁজে পাওয়ার ব্যবস্থা করা।
\item text, image, link, list, table ও form-এর সঠিক ব্যবহার নিশ্চিত করা।
\item বিভিন্ন browser ও device-এ web page ঠিকভাবে দেখা যায় এমনভাবে তৈরি করা।
\item প্রতিষ্ঠান, পণ্য, সেবা বা ব্যক্তিগত পরিচিতি online-এ প্রকাশ করা।
\item visitor বা user যেন সহজে contact, download বা service নিতে পারে সে ব্যবস্থা রাখা।
\end{itemize}
\end{keypointbox}

\subsection{ওয়েব ডিজাইনের সাথে সম্পর্কিত মৌলিক ধারণা}

ওয়েব ডিজাইন ভালোভাবে বুঝতে হলে কয়েকটি basic term জানা প্রয়োজন।

\subsubsection{ওয়েবপেজ}

\begin{definitionbox}{ওয়েবপেজ}
ওয়েবপেজ হলো এমন একটি web document, যা web browser দিয়ে দেখা যায় এবং সাধারণত HTML দিয়ে তৈরি করা হয়।
\end{definitionbox}

একটি ওয়েবসাইটের স্বতন্ত্র যেকোনো page-কে ওয়েবপেজ বলা যায়। যেমন, একটি কলেজ ওয়েবসাইটে Home, About, Notice, Admission, Result, Contact - প্রতিটি আলাদা ওয়েবপেজ হতে পারে।

\begin{examplebox}{উদাহরণ}
\texttt{notice.html} একটি notice page হতে পারে, যেখানে কলেজের notice list দেওয়া থাকে। \texttt{result.html} একটি result page হতে পারে, যেখানে student result table দেওয়া থাকে।
\end{examplebox}

\begin{keypointbox}{History Highlight: Webpage-এর শুরু}
\begin{itemize}
\item Webpage ধারণার জন্ম World Wide Web-এর সাথে সম্পর্কিত।
\item \textbf{Tim Berners-Lee} ১৯৮৯ সালে CERN-এ World Wide Web-এর ধারণা দেন।
\item ১৯৯০ সালে তিনি প্রথম web server, প্রথম web browser/editor এবং প্রথম website তৈরির কাজ করেন।
\item প্রথম web page-গুলো ছিল simple HTML document, যেখানে text ও hyperlink ছিল।
\item Webpage-এর মূল শক্তি ছিল hypertext link - এক page থেকে অন্য page-এ যাওয়ার সুবিধা।
\end{itemize}
\end{keypointbox}

\subsubsection{ওয়েবসাইট}

\begin{definitionbox}{ওয়েবসাইট}
একই domain name বা web address-এর অধীনে থাকা সম্পর্কিত একাধিক ওয়েবপেজের সমষ্টিকে ওয়েবসাইট বলা হয়।
\end{definitionbox}

কোনো ওয়েবসাইটে প্রবেশ করলে সাধারণত প্রথম যে page দেখা যায় তাকে homepage বলা হয়। Homepage থেকে About, Notice, Result, Contact ইত্যাদি page-এ যাওয়ার link থাকে।

\begin{examplebox}{উদাহরণ}
\texttt{www.educationboard.gov.bd} একটি ওয়েবসাইট। এর অধীনে notice page, result page, circular page ইত্যাদি থাকতে পারে।
\end{examplebox}

\subsubsection{ওয়েব সার্ভার}

\begin{definitionbox}{ওয়েব সার্ভার}
যে server-এ ওয়েবসাইট বা ওয়েবপেজের file সংরক্ষিত থাকে এবং browser request করলে সেই file পাঠায়, তাকে ওয়েব সার্ভার বলা হয়।
\end{definitionbox}

ওয়েব সার্ভারে HTML file, CSS file, image, video, PDF, database বা website-এর অন্যান্য resource রাখা থাকতে পারে। ব্যবহারকারী browser-এ website address লিখলে browser সেই server-এর কাছে request পাঠায়।

\subsubsection{ওয়েব ব্রাউজার}

\begin{definitionbox}{ওয়েব ব্রাউজার}
ওয়েব ব্রাউজার হলো এমন application software, যার মাধ্যমে ব্যবহারকারী ওয়েবপেজ দেখতে, hyperlink follow করতে, file download করতে এবং website ব্যবহার করতে পারে।
\end{definitionbox}

\begin{examplebox}{উদাহরণ}
Google Chrome, Mozilla Firefox, Microsoft Edge, Safari ও Opera হলো web browser-এর উদাহরণ।
\end{examplebox}

\begin{keypointbox}{Browser-এর কাজ}
\begin{itemize}
\item URL অনুযায়ী server-এ request পাঠানো।
\item HTML, CSS ও image render করে web page দেখানো।
\item hyperlink-এর মাধ্যমে এক page থেকে অন্য page-এ যেতে সাহায্য করা।
\item file download, form submit ও website interaction করতে সাহায্য করা।
\end{itemize}
\end{keypointbox}

\subsection{ওয়েব ডিজাইন কীভাবে কাজ করে?}

একটি ওয়েবপেজ তৈরি করার পর সেটি server-এ রাখা হয়। ব্যবহারকারী browser-এ website address বা URL লিখলে browser web server-এ request পাঠায়। Server সেই request অনুযায়ী web page-এর file পাঠায়। এরপর browser HTML/CSS/image file interpret করে user-এর সামনে webpage আকারে দেখায়।

\begin{enumerate}
\item User browser-এ URL লিখে।
\item Browser server-এ request পাঠায়।
\item Web server requested file খুঁজে response পাঠায়।
\item Browser HTML, CSS ও image process করে।
\item User screen-এ web page output দেখা যায়।
\end{enumerate}

\subsection{ওয়েব ডিজাইনে HTML-এর ভূমিকা}

HTML-এর পূর্ণরূপ হলো \textbf{HyperText Markup Language}। এটি ওয়েবপেজ তৈরির মৌলিক ভাষা। HTML ব্যবহার করে web page-এর structure তৈরি করা হয় - কোথায় heading থাকবে, কোথায় paragraph থাকবে, কোথায় image, list, table, form বা hyperlink থাকবে তা HTML tag দিয়ে নির্ধারণ করা হয়।

\begin{keypointbox}{HTML, CSS ও JavaScript}
\begin{itemize}
\item \textbf{HTML} web page-এর structure বা কাঠামো তৈরি করে।
\item \textbf{CSS} web page-এর design, color, font, spacing ও layout সুন্দর করে।
\item \textbf{JavaScript} web page-এ dynamic behavior ও interactivity যোগ করে।
\end{itemize}
\end{keypointbox}

\begin{boardbox}{Exam-standard line}
HTML ওয়েবপেজের কাঠামো তৈরি করে, CSS ওয়েবপেজকে সুন্দর করে এবং JavaScript ওয়েবপেজকে dynamic বা interactive করে।
\end{boardbox}

\subsection{ওয়েব ডিজাইনের সুবিধা}

ওয়েব ডিজাইনের মাধ্যমে ব্যক্তি বা প্রতিষ্ঠান সহজে তাদের তথ্য বিশ্বব্যাপী প্রকাশ করতে পারে। শিক্ষার্থী, অভিভাবক, customer বা visitor যেকোনো স্থান থেকে প্রয়োজনীয় তথ্য দেখতে পারে।

\begin{itemize}
\item তথ্য দ্রুত প্রকাশ করা যায়।
\item যেকোনো স্থান থেকে তথ্য দেখা যায়।
\item প্রতিষ্ঠান বা ব্যবসার পরিচিতি বাড়ে।
\item online service দেওয়া যায়।
\item শিক্ষার্থীরা online class, note, result, routine ও notice পেতে পারে।
\item customer বা visitor সহজে যোগাযোগ করতে পারে।
\item কম খরচে বিশ্বব্যাপী প্রচার করা যায়।
\item প্রয়োজনীয় file, PDF, form বা notice download করা যায়।
\end{itemize}

\subsection{ওয়েব ডিজাইনের ব্যবহারক্ষেত্র}

\begin{itemize}
\item \textbf{শিক্ষা প্রতিষ্ঠান:} notice, result, admission, routine, teacher information।
\item \textbf{ব্যবসা প্রতিষ্ঠান:} product, service, order, payment information।
\item \textbf{সংবাদমাধ্যম:} online news portal।
\item \textbf{সরকারি সেবা:} নাগরিক সেবা, আবেদন, তথ্য প্রকাশ।
\item \textbf{স্বাস্থ্যসেবা:} appointment, doctor information, health notice।
\item \textbf{ব্যক্তিগত portfolio:} নিজের কাজ, skill, project দেখানো।
\item \textbf{E-commerce:} online shopping, product catalog, cart।
\item \textbf{Banking:} online banking information ও service notice।
\end{itemize}

\subsection{Static ও Dynamic ওয়েবসাইট: প্রাথমিক ধারণা}

ওয়েবসাইটের content যদি সাধারণত fixed থাকে এবং user অনুযায়ী স্বয়ংক্রিয়ভাবে পরিবর্তিত না হয়, তাহলে সেটি static website। আর content যদি user input, database বা server-side program অনুযায়ী পরিবর্তিত হয়, তাহলে সেটি dynamic website। এই অধ্যায়ে পরের টপিকে static, dynamic, local ও remote website আলাদা করে বিস্তারিত আলোচনা করা হয়েছে।

\begin{keypointbox}{মনে রাখো}
এই টপিকে শুধু ধারণা বোঝানো হলো। পরীক্ষার তুলনামূলক উত্তর লেখার জন্য ``ওয়েবসাইটের প্রকারভেদ'' টপিকের table অনুসরণ করবে।
\end{keypointbox}

\subsection{Simple HTML Skeleton}

নিচের code টি একটি simple web page-এর basic structure দেখায়।

\begin{examplebox}{Code}
\begin{verbatim}
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>Example College</title>
</head>
<body>
  <h1>Example College</h1>
  <p>Welcome to our college website.</p>
  <a href="notice.html">Notice</a>
  <br>
  <img src="images/college.jpg" alt="College" width="300">
</body>
</html>
\end{verbatim}
\end{examplebox}

\begin{examplebox}{Output ধারণা}
Browser tab-এ \texttt{Example College} title দেখা যাবে। Web page-এর body অংশে \textbf{Example College} heading, একটি welcome paragraph, একটি Notice link এবং একটি college image দেখা যাবে।
\end{examplebox}

\begin{keypointbox}{Code explanation}
\begin{itemize}
\item \texttt{<!DOCTYPE html>} document-টি HTML5 standard অনুসরণ করে তা বোঝায়।
\item \texttt{<title>} browser tab-এর title নির্ধারণ করে।
\item \texttt{<h1>} বড় heading তৈরি করে।
\item \texttt{<p>} paragraph তৈরি করে।
\item \texttt{<a href="notice.html">} internal hyperlink তৈরি করে।
\item \texttt{<img>} web page-এ image দেখায়।
\end{itemize}
\end{keypointbox}

\subsection{পরীক্ষার জন্য সংক্ষিপ্ত উত্তর}

\subsubsection{প্রশ্ন: ওয়েব ডিজাইন কী?}

\begin{boardbox}{উত্তর}
ওয়েবসাইট বা ওয়েবপেজের তথ্য, ছবি, লিংক, টেবিল, মেনু ও অন্যান্য উপাদানকে নির্দিষ্ট কাঠামো অনুযায়ী সুন্দর ও ব্যবহারবান্ধবভাবে সাজানোর প্রক্রিয়াকে ওয়েব ডিজাইন বলে। এর মাধ্যমে internet-এ প্রদর্শনের উপযোগী web page তৈরি করা হয়।
\end{boardbox}

\subsubsection{প্রশ্ন: ওয়েব ডিজাইন কেন গুরুত্বপূর্ণ?}

\begin{boardbox}{উত্তর}
ওয়েব ডিজাইন গুরুত্বপূর্ণ, কারণ এর মাধ্যমে কোনো ব্যক্তি বা প্রতিষ্ঠান তাদের তথ্য, সেবা বা পণ্য internet-এর মাধ্যমে আকর্ষণীয় ও সহজভাবে ব্যবহারকারীর কাছে উপস্থাপন করতে পারে। ভালো ওয়েব ডিজাইন তথ্য খুঁজে পাওয়া সহজ করে, ওয়েবসাইটকে user-friendly করে এবং প্রতিষ্ঠানের পরিচিতি বৃদ্ধি করে।
\end{boardbox}
