\section{ডেটা ট্রান্সমিশন মেথড ও মোড}

\begin{learningbox}
এই টপিক শেষে শিক্ষার্থী পারবে:
\begin{itemize}
\item ডেটা ট্রান্সমিশন কী তা ব্যাখ্যা করতে।
\item Parallel transmission ও Serial transmission-এর পার্থক্য বুঝতে।
\item Asynchronous, Synchronous ও Isochronous transmission ব্যাখ্যা করতে।
\item Simplex, Half-duplex ও Full-duplex transmission mode তুলনা করতে।
\item Unicast, Broadcast ও Multicast data delivery mode শনাক্ত করতে।
\item বোর্ড ও ভর্তি পরীক্ষার প্রশ্নে transmission method/mode প্রয়োগ করতে।
\end{itemize}
\end{learningbox}

\subsection{বাস্তব জীবনের শুরু}

আমরা যখন mobile দিয়ে message পাঠাই, computer থেকে file transfer করি, video call করি বা online class করি—তখন data এক device থেকে অন্য device-এ যায়। কিন্তু data সবসময় একইভাবে যায় না। কখনো data একসাথে অনেক bit আকারে যায়, কখনো একটির পর একটি bit যায়, আবার কখনো live audio-video stream-এর মতো নির্দিষ্ট সময় বজায় রেখে যায়।

এই data যাওয়ার পদ্ধতিকে বলা হয় \textbf{Data Transmission Method}। আর data কোন direction-এ যাচ্ছে—একদিকে, পালাক্রমে, নাকি একই সাথে দুইদিকে—এটি বোঝায় \textbf{Transmission Mode}।

\begin{keypointbox}{মূল ধারণা}
Data transmission বুঝতে তিনটি প্রশ্ন মনে রাখো:
\begin{itemize}
\item Data কীভাবে যাচ্ছে? — Parallel না Serial?
\item Data কখন/কোন নিয়মে যাচ্ছে? — Asynchronous, Synchronous না Isochronous?
\item Data কোন direction-এ যাচ্ছে? — Simplex, Half-duplex না Full-duplex?
\end{itemize}
\end{keypointbox}

\subsection{ডেটা ট্রান্সমিশন}

\begin{definitionbox}{ডেটা ট্রান্সমিশন}
এক device থেকে অন্য device-এ communication medium-এর মাধ্যমে data পাঠানোর প্রক্রিয়াকে data transmission বলা হয়।
\end{definitionbox}

Data transmission সাধারণত signal-এর মাধ্যমে সম্পন্ন হয়। Computer data binary আকারে অর্থাৎ 0 ও 1 হিসেবে সংরক্ষণ ও process করে। এই 0 ও 1 communication medium দিয়ে পাঠানোর সময় electrical signal, optical signal বা electromagnetic signal হিসেবে transmitted হয়।

\end{boardbox}

\subsection{ডেটা ট্রান্সমিশন মেথড}

Data transmission method মূলত দুই ধরনের:

\begin{itemize}
\item Parallel Transmission
\item Serial Transmission
\end{itemize}

\subsection{Parallel Transmission}

\begin{definitionbox}{Parallel Transmission}
যে transmission পদ্ধতিতে একাধিক bit একই সময়ে আলাদা আলাদা channel বা line দিয়ে পাঠানো হয়, তাকে parallel transmission বলা হয়।
\end{definitionbox}

Parallel transmission-এ 8 bit data পাঠাতে চাইলে 8টি আলাদা line ব্যবহার করা যেতে পারে। ফলে একই সময়ে অনেক bit পাঠানো যায়। তাই short distance-এ এটি দ্রুত। তবে line বেশি লাগার কারণে cost বেশি হয় এবং দূরত্ব বাড়লে synchronization problem হতে পারে।

\begin{keypointbox}{Parallel transmission-এর বৈশিষ্ট্য}
\begin{itemize}
\item একাধিক bit একই সময়ে পাঠানো হয়।
\item একাধিক channel বা wire প্রয়োজন হয়।
\item short distance communication-এ কার্যকর।
\item speed বেশি হলেও cost বেশি।
\item দূরত্ব বেশি হলে timing mismatch বা synchronization problem হতে পারে।
\end{itemize}
\end{keypointbox}

\begin{examplebox}{উদাহরণ}
Computer-এর internal bus বা পুরনো printer port-এ parallel transmission ব্যবহৃত হতো। কারণ এগুলো সাধারণত short distance-এর মধ্যে data transfer করত।
\end{examplebox}

\subsection{Serial Transmission}

\begin{definitionbox}{Serial Transmission}
যে transmission পদ্ধতিতে data bit-by-bit অর্থাৎ একটির পর একটি bit একই channel দিয়ে পাঠানো হয়, তাকে serial transmission বলা হয়।
\end{definitionbox}

Serial transmission-এ line কম লাগে, তাই cost কম। Long distance communication-এর জন্য এটি বেশি উপযোগী। Modern communication system, USB, Ethernet, mobile network এবং internet communication-এ serial transmission-এর ধারণা গুরুত্বপূর্ণ।

\begin{keypointbox}{Serial transmission-এর বৈশিষ্ট্য}
\begin{itemize}
\item data একটির পর একটি bit হিসেবে পাঠানো হয়।
\item সাধারণত একটি channel বা কম line প্রয়োজন হয়।
\item long distance communication-এর জন্য উপযোগী।
\item cost তুলনামূলক কম।
\item speed নির্ভর করে channel capacity, clocking, protocol ও technology-এর উপর।
\end{itemize}
\end{keypointbox}

\begin{examplebox}{উদাহরণ}
USB, Ethernet communication, modem communication এবং long-distance network communication-এ serial transmission বহুল ব্যবহৃত।
\end{examplebox}

\subsection{Parallel Transmission ও Serial Transmission-এর পার্থক্য}

\begin{center}
\begin{tabular}{|p{0.22\textwidth}|p{0.32\textwidth}|p{0.32\textwidth}|}
\hline
\textbf{বিষয়} & \textbf{Parallel Transmission} & \textbf{Serial Transmission} \\

\hline
Line/Channel & একাধিক line প্রয়োজন & সাধারণত একটি line বা কম line প্রয়োজন \\

\end{tabular}
\end{center}

\begin{mistakebox}{Common mistake}
অনেকে মনে করে parallel transmission সবসময় serial transmission-এর চেয়ে ভালো। আসলে short distance-এ parallel দ্রুত হতে পারে, কিন্তু long distance ও modern networking-এ serial transmission বেশি practical ও reliable।
\end{mistakebox}

\subsection{Serial Transmission-এর ধরন}

Serial transmission সাধারণত তিন ধরনের:

\begin{itemize}
\item Asynchronous Transmission
\item Synchronous Transmission
\item Isochronous Transmission
\end{itemize}

\subsection{Asynchronous Transmission}

\begin{definitionbox}{Asynchronous Transmission}
যে serial transmission পদ্ধতিতে data character বা small unit আকারে পাঠানো হয় এবং প্রতিটি unit-এর শুরুতে start bit ও শেষে stop bit থাকে, তাকে asynchronous transmission বলা হয়।
\end{definitionbox}

Asynchronous transmission-এ sender ও receiver-এর clock পুরো সময় synchronized থাকতে হয় না। প্রতিটি character বা byte আলাদাভাবে পাঠানো হয়। Receiver start bit দেখে বুঝতে পারে data শুরু হয়েছে এবং stop bit দেখে বুঝতে পারে data শেষ হয়েছে।

}{
\fbox{
\parbox{0.90\textwidth}{

\vspace{0.7cm}

\begin{keypointbox}{Asynchronous transmission-এর বৈশিষ্ট্য}
\begin{itemize}
\item data ছোট unit বা character আকারে পাঠানো হয়।
\item প্রতিটি unit-এর সাথে start bit ও stop bit থাকে।
\item sender ও receiver-এর continuous clock synchronization দরকার হয় না।
\item data কম হলে বা অনিয়মিতভাবে data পাঠাতে হলে উপযোগী।
\item extra start/stop bit থাকার কারণে overhead বেশি হতে পারে।
\end{itemize}
\end{keypointbox}

\begin{examplebox}{উদাহরণ}
Keyboard input, পুরনো serial communication এবং কম পরিমাণ অনিয়মিত data পাঠানোর ক্ষেত্রে asynchronous transmission ব্যবহৃত হতে পারে।
\end{examplebox}

\subsection{Synchronous Transmission}

\begin{definitionbox}{Synchronous Transmission}
যে serial transmission পদ্ধতিতে sender ও receiver clock synchronization বজায় রেখে data block বা frame আকারে পাঠায়, তাকে synchronous transmission বলা হয়।
\end{definitionbox}

Synchronous transmission-এ data character-by-character না গিয়ে বড় block বা frame হিসেবে যায়। এখানে start/stop bit প্রতিটি character-এর সাথে থাকে না। ফলে large amount data দ্রুত ও দক্ষতার সাথে পাঠানো যায়। তবে sender ও receiver-এর timing বা clock synchronization গুরুত্বপূর্ণ।

\begin{keypointbox}{Synchronous transmission-এর বৈশিষ্ট্য}
\begin{itemize}
\item data block বা frame আকারে পাঠানো হয়।
\item sender ও receiver-এর clock synchronization প্রয়োজন।
\item বড় পরিমাণ data দ্রুত transfer-এর জন্য উপযোগী।
\item overhead তুলনামূলক কম।
\item network communication-এ বেশি কার্যকর।
\end{itemize}
\end{keypointbox}

\begin{examplebox}{উদাহরণ}
Network communication, high-speed data transfer, LAN communication ইত্যাদিতে synchronous transmission-এর ধারণা ব্যবহৃত হয়।
\end{examplebox}

\subsection{Isochronous Transmission}

\begin{definitionbox}{Isochronous Transmission}
যে transmission পদ্ধতিতে data নির্দিষ্ট সময় interval বজায় রেখে continuous stream হিসেবে পাঠানো হয়, তাকে isochronous transmission বলা হয়।
\end{definitionbox}

Isochronous transmission বিশেষভাবে audio ও video communication-এর জন্য গুরুত্বপূর্ণ। এখানে শুধু data পৌঁছানোই যথেষ্ট নয়; data নির্দিষ্ট সময়ের মধ্যে পৌঁছাতে হবে। কারণ video call, live class বা streaming-এ delay বেশি হলে sound ও video mismatch হতে পারে।

\begin{keypointbox}{Isochronous transmission-এর বৈশিষ্ট্য}
\begin{itemize}
\item continuous data stream পাঠানো হয়।
\item নির্দিষ্ট সময় interval বজায় রাখা গুরুত্বপূর্ণ।
\item audio, video ও real-time communication-এর জন্য উপযোগী।
\item delay বেশি হলে communication quality কমে যায়।
\item live streaming, video conference, online class-এ এর ধারণা গুরুত্বপূর্ণ।
\end{itemize}
\end{keypointbox}

\begin{examplebox}{উদাহরণ}
Video call, live online class, IP TV, live audio-video streaming—এসব ক্ষেত্রে timing খুব গুরুত্বপূর্ণ, তাই isochronous transmission-এর ধারণা প্রযোজ্য।
\end{examplebox}

Asynchronous transmission-এর প্রশ্নে start bit ও stop bit অবশ্যই লিখতে হবে। Synchronous transmission-এর ক্ষেত্রে block/frame ও clock synchronization লিখতে হবে। Isochronous transmission-এর ক্ষেত্রে real-time audio-video এবং fixed time interval লিখলে উত্তর পূর্ণাঙ্গ হয়।
\end{boardbox}

\subsection{ডেটা ট্রান্সমিশন মোড}

\begin{definitionbox}{Transmission Mode}
Data sender ও receiver-এর মধ্যে কোন direction-এ আদান-প্রদান হবে, সেই পদ্ধতিকে transmission mode বলা হয়।
\end{definitionbox}

Transmission mode তিন ধরনের:

\begin{itemize}
\item Simplex
\item Half-duplex
\item Full-duplex
\end{itemize}

\subsection{Simplex Mode}

\begin{definitionbox}{Simplex Mode}
যে transmission mode-এ data শুধু এক direction-এ যায়, অর্থাৎ sender শুধু পাঠায় এবং receiver শুধু গ্রহণ করে, তাকে simplex mode বলা হয়।
\end{definitionbox}

Simplex mode-এ receiver data পাঠাতে পারে না। এটি one-way communication system। Broadcasting system-এ এটি বেশি দেখা যায়।

\begin{examplebox}{উদাহরণ}
Radio broadcast, TV broadcast, keyboard থেকে computer-এ input, monitor-এ output display—এসব simplex communication-এর উদাহরণ।
\end{examplebox}

\subsection{Half-duplex Mode}

\begin{definitionbox}{Half-duplex Mode}
যে transmission mode-এ data দুই direction-এ যেতে পারে, কিন্তু একই সময়ে দুই direction-এ যেতে পারে না, তাকে half-duplex mode বলা হয়।
\end{definitionbox}

Half-duplex mode-এ উভয় পক্ষ data পাঠাতে ও গ্রহণ করতে পারে, কিন্তু একসাথে নয়। একজন কথা বললে অন্যজন অপেক্ষা করে।

\begin{examplebox}{উদাহরণ}
Walkie-talkie half-duplex communication-এর সহজ উদাহরণ। একজন কথা বলার সময় অন্যজন শুনে, পরে অন্যজন কথা বলে।
\end{examplebox}

\subsection{Full-duplex Mode}

\begin{definitionbox}{Full-duplex Mode}
যে transmission mode-এ data একই সময়ে দুই direction-এ আদান-প্রদান করা যায়, তাকে full-duplex mode বলা হয়।
\end{definitionbox}

Full-duplex mode আধুনিক communication-এ খুব গুরুত্বপূর্ণ, কারণ এতে sender ও receiver একই সময়ে data পাঠাতে ও গ্রহণ করতে পারে। ফলে communication দ্রুত ও স্বাভাবিক হয়।

\begin{examplebox}{উদাহরণ}
Mobile phone call, video call এবং modern network communication full-duplex communication-এর উদাহরণ।
\end{examplebox}

\begin{mistakebox}{Common mistake}
Half-duplex ও full-duplex দুটোতেই data দুইদিকে যেতে পারে। পার্থক্য হলো half-duplex-এ একই সময়ে দুইদিকে যায় না, কিন্তু full-duplex-এ একই সময়ে দুইদিকে data আদান-প্রদান হয়।
\end{mistakebox}

\subsection{Data Delivery Mode: Unicast, Broadcast ও Multicast}

Data transmission-এ কখনো data এক receiver-এর কাছে যায়, কখনো network-এর সব receiver-এর কাছে যায়, আবার কখনো নির্দিষ্ট group-এর কাছে যায়। এই ভিত্তিতে delivery mode তিন ধরনের:

\begin{itemize}
\item Unicast
\item Broadcast
\item Multicast
\end{itemize}

\subsection{Unicast}

\begin{definitionbox}{Unicast}
এক sender থেকে এক নির্দিষ্ট receiver-এর কাছে data পাঠানোর পদ্ধতিকে unicast বলা হয়।
\end{definitionbox}

\begin{examplebox}{উদাহরণ}
একজন বন্ধুকে email পাঠানো, এক ব্যক্তিকে WhatsApp message পাঠানো, একটি device থেকে নির্দিষ্ট server-এ request পাঠানো—unicast communication-এর উদাহরণ।
\end{examplebox}

\subsection{Broadcast}

\begin{definitionbox}{Broadcast}
এক sender থেকে network-এর সকল receiver-এর কাছে data পাঠানোর পদ্ধতিকে broadcast বলা হয়।
\end{definitionbox}

\begin{examplebox}{উদাহরণ}
TV broadcast, radio broadcast এবং local network-এ broadcast message—broadcast communication-এর উদাহরণ।
\end{examplebox}

\subsection{Multicast}

\begin{definitionbox}{Multicast}
এক sender থেকে নির্দিষ্ট একটি receiver group-এর কাছে data পাঠানোর পদ্ধতিকে multicast বলা হয়।
\end{definitionbox}

\begin{examplebox}{উদাহরণ}
একটি online class link নির্দিষ্ট group-এর শিক্ষার্থীদের কাছে পাঠানো, group video streaming, IPTV multicast—multicast communication-এর উদাহরণ।
\end{examplebox}

\subsection{Exam-style প্রশ্ন ও উত্তর}

\begin{boardbox}{সংক্ষিপ্ত প্রশ্ন: Parallel transmission কী?}
যে transmission পদ্ধতিতে একাধিক bit একই সময়ে আলাদা আলাদা line বা channel দিয়ে পাঠানো হয়, তাকে parallel transmission বলা হয়। এতে speed বেশি হলেও একাধিক line লাগে, তাই cost বেশি এবং long distance-এ synchronization problem হতে পারে।
\end{boardbox}

\begin{boardbox}{সংক্ষিপ্ত প্রশ্ন: Serial transmission কেন long distance-এ উপযোগী?}
Serial transmission-এ data bit-by-bit একই channel দিয়ে পাঠানো হয়। এতে line কম লাগে, cost কম হয় এবং long distance communication সহজ হয়। তাই modern network, USB, Ethernet ও internet communication-এ serial transmission বেশি ব্যবহৃত হয়।
\end{boardbox}

\begin{boardbox}{সংক্ষিপ্ত প্রশ্ন: Asynchronous transmission কী?}
যে serial transmission পদ্ধতিতে data character বা small unit আকারে পাঠানো হয় এবং প্রতিটি unit-এর শুরুতে start bit ও শেষে stop bit থাকে, তাকে asynchronous transmission বলা হয়। এতে sender ও receiver-এর continuous clock synchronization প্রয়োজন হয় না।
\end{boardbox}

\begin{boardbox}{পার্থক্য: Simplex ও Full-duplex}
Simplex mode-এ data শুধু একদিকে যায়; receiver data পাঠাতে পারে না। কিন্তু full-duplex mode-এ একই সময়ে দুই direction-এ data আদান-প্রদান করা যায়। TV broadcast simplex-এর উদাহরণ, আর mobile phone call full-duplex-এর উদাহরণ।
\end{boardbox}

\subsection{১ মিনিটে রিভিশন}

\begin{recapbox}
\begin{itemize}
\item Data transmission হলো এক device থেকে অন্য device-এ data পাঠানোর প্রক্রিয়া।
\item Parallel transmission-এ একাধিক bit একই সময়ে যায়।
\item Serial transmission-এ bit একটির পর একটি যায়।
\item Asynchronous transmission-এ start bit ও stop bit থাকে।
\item Synchronous transmission-এ data block/frame আকারে যায়।
\item Isochronous transmission-এ fixed time interval বজায় রাখা গুরুত্বপূর্ণ।
\item Simplex হলো one-way communication।
\item Half-duplex হলো two-way but one at a time।
\item Full-duplex হলো two-way at the same time।
\item Unicast one-to-one, Broadcast one-to-all, Multicast one-to-selected group।
\end{itemize}
\end{recapbox}
