\section{কমিউনিকেশন সিস্টেম ও ব্যান্ডউইডথ}

\begin{learningbox}
এই টপিক শেষে শিক্ষার্থী পারবে:
\begin{itemize}
\item কমিউনিকেশন, কমিউনিকেশন সিস্টেম ও ডেটা কমিউনিকেশন ব্যাখ্যা করতে।
\item কমিউনিকেশন সিস্টেমের প্রধান উপাদানগুলো শনাক্ত করতে।
\item Data, Information ও Signal-এর পার্থক্য বুঝতে।
\item Analog signal ও Digital signal-এর বৈশিষ্ট্য তুলনা করতে।
\item Bandwidth, Data rate ও Latency-এর সম্পর্ক ব্যাখ্যা করতে।
\item Narrow band, Voice band ও Broadband সম্পর্কে exam-standard উত্তর লিখতে।
\item Bandwidth calculation করতে।
\end{itemize}
\end{learningbox}

\subsection{বাস্তব জীবনের শুরু}

ধরা যাক, একজন শিক্ষার্থী laptop দিয়ে তার বন্ধুকে একটি email পাঠাল। Email পাঠানোর সময় শিক্ষার্থীর laptop হলো \textbf{Sender}, email-এর লেখা ও attachment হলো \textbf{Message/Data}, internet connection হলো \textbf{Medium}, আর email পাঠানোর নিয়ম হলো \textbf{Protocol}। বন্ধুর mobile বা laptop হলো \textbf{Receiver/Destination}।

এই পুরো প্রক্রিয়াটিই একটি communication system-এর বাস্তব উদাহরণ। অর্থাৎ ICT-তে communication system মানে শুধু কথা বলা নয়; বরং যেকোনো data, text, image, audio, video বা file এক device থেকে অন্য device-এ পাঠানোর সম্পূর্ণ ব্যবস্থা।

\begin{keypointbox}{মূল ধারণা}
Communication system বুঝতে ৫টি প্রশ্ন মনে রাখলেই concept পরিষ্কার হয়:
\begin{itemize}
\item কে পাঠাচ্ছে? — Sender
\item কী পাঠাচ্ছে? — Message/Data
\item কোন পথে পাঠাচ্ছে? — Medium
\item কোন নিয়মে পাঠাচ্ছে? — Protocol
\item কে গ্রহণ করছে? — Receiver
\end{itemize}
\end{keypointbox}

\subsection{কমিউনিকেশন}

\begin{definitionbox}{কমিউনিকেশন}
এক ব্যক্তি, যন্ত্র বা সিস্টেম থেকে অন্য ব্যক্তি, যন্ত্র বা সিস্টেমে তথ্য, ধারণা, বার্তা বা ডেটা আদান-প্রদানের প্রক্রিয়াকে কমিউনিকেশন বলা হয়।
\end{definitionbox}

কমিউনিকেশন মানুষের মধ্যে হতে পারে, আবার যন্ত্রের মধ্যেও হতে পারে। যেমন—মানুষ mobile phone দিয়ে কথা বলে, computer server থেকে website data পাঠায়, router data packet forward করে, cloud server file সংরক্ষণ ও সরবরাহ করে। ICT-তে communication সাধারণত signal-এর মাধ্যমে সম্পন্ন হয়।

\begin{examplebox}{বাস্তব উদাহরণ}
Mobile call, SMS, email, online class, video conference, website browsing, ATM transaction, bKash/Nagad transaction, cloud storage-এ file upload—সবই communication-এর উদাহরণ।
\end{examplebox}

\begin{boardbox}{বোর্ড ফোকাস}
“Communication কী?” প্রশ্নে শুধু তথ্য আদান-প্রদান বললে উত্তর ছোট হয়ে যায়। ভালো উত্তর লিখতে হলে sender, receiver, message এবং medium-এর ধারণা যুক্ত করা উচিত।
\end{boardbox}

\subsection{কমিউনিকেশন সিস্টেম}

\begin{definitionbox}{কমিউনিকেশন সিস্টেম}
Sender থেকে receiver-এর কাছে message বা data পাঠানোর জন্য sender, receiver, message, medium, protocol এবং প্রয়োজনীয় device নিয়ে গঠিত সম্পূর্ণ ব্যবস্থাকে communication system বলা হয়।
\end{definitionbox}

একটি communication system সফল হতে হলে data সঠিকভাবে, নির্দিষ্ট সময়ে এবং নির্ভরযোগ্যভাবে receiver-এর কাছে পৌঁছাতে হবে। Data যদি ভুলভাবে পৌঁছে, দেরিতে পৌঁছে, মাঝপথে হারিয়ে যায় বা unauthorized ব্যক্তি access করে, তাহলে communication system কার্যকর হয় না।

\subsection{কমিউনিকেশন সিস্টেমের কার্যপ্রণালি}

কমিউনিকেশন সিস্টেমে sender প্রথমে message তৈরি করে। Message সরাসরি medium দিয়ে যেতে পারে না; তাই অনেক ক্ষেত্রে message-কে signal-এ রূপান্তর করা হয়। এই কাজ encoder বা transmitter করে। এরপর signal medium দিয়ে receiver-এর দিকে যায়। Receiver side-এ decoder signal-কে আবার বুঝতে পারার মতো data বা information-এ রূপান্তর করে।

\begin{examplebox}{Mobile message example}
যখন একজন শিক্ষার্থী mobile থেকে message পাঠায়, তখন message digital data হিসেবে mobile network-এ যায়। Mobile network সেই data signal আকারে বহন করে। Receiver-এর mobile signal গ্রহণ করে আবার readable message হিসেবে দেখায়।
\end{examplebox}

\begin{mistakebox}{Common mistake}
অনেকে Sender এবং Transmitter-কে একই মনে করে। Sender হলো যে data পাঠায়; Transmitter হলো যে data বা message-কে transmission-এর উপযোগী signal-এ রূপান্তর করে পাঠায়।
\end{mistakebox}

\subsection{ডেটা কমিউনিকেশন}

\begin{definitionbox}{ডেটা কমিউনিকেশন}
Digital বা analog data-কে নির্দিষ্ট communication medium ও protocol ব্যবহার করে এক device থেকে অন্য device-এ পাঠানো ও গ্রহণ করার প্রক্রিয়াকে data communication বলা হয়।
\end{definitionbox}

Data communication আধুনিক ICT-এর ভিত্তি। Internet, LAN, WAN, mobile network, online banking, cloud computing, e-mail, video call—সবকিছু data communication-এর উপর নির্ভরশীল। Computer network-এ data communication সাধারণত digital signal, protocol এবং network device-এর সাহায্যে সম্পন্ন হয়।

\begin{keypointbox}{ভালো ডেটা কমিউনিকেশনের বৈশিষ্ট্য}
\begin{itemize}
\item \textbf{Delivery}: data সঠিক receiver-এর কাছে পৌঁছাতে হবে।
\item \textbf{Accuracy}: data ভুল বা বিকৃত হওয়া যাবে না।
\item \textbf{Timeliness}: data প্রয়োজনীয় সময়ের মধ্যে পৌঁছাতে হবে।
\item \textbf{Security}: unauthorized access থেকে data নিরাপদ থাকতে হবে।
\item \textbf{Reliability}: data loss কম হতে হবে এবং error হলে correction-এর ব্যবস্থা থাকতে হবে।
\end{itemize}
\end{keypointbox}

\begin{boardbox}{Exam-standard answer}
ডেটা কমিউনিকেশন হলো এমন একটি প্রক্রিয়া যার মাধ্যমে digital বা analog data নির্দিষ্ট medium ও protocol ব্যবহার করে এক device থেকে অন্য device-এ পাঠানো ও গ্রহণ করা হয়। এতে sender, receiver, message, medium এবং protocol থাকে। যেমন—email পাঠানো, mobile message, online class ইত্যাদি।
\end{boardbox}

\subsection{Data, Information ও Signal}

\begin{definitionbox}{Data}
Data হলো কাঁচা fact, symbol, number, text, image, audio বা video, যা processing-এর আগে বা পরে communication-এ ব্যবহৃত হতে পারে।
\end{definitionbox}

\begin{definitionbox}{Information}
Data প্রক্রিয়াকরণের ফলে যে অর্থপূর্ণ ফলাফল পাওয়া যায়, তাকে information বলা হয়।
\end{definitionbox}

\begin{definitionbox}{Signal}
Signal হলো electrical, optical বা electromagnetic পরিবর্তন, যার মাধ্যমে data এক স্থান থেকে অন্য স্থানে বহন করা হয়।
\end{definitionbox}

\begin{center}
\begin{tabular}{|p{0.20\textwidth}|p{0.22\textwidth}|p{0.22\textwidth}|p{0.22\textwidth}|}
\hline
\textbf{বিষয়} & \textbf{Data} & \textbf{Information} & \textbf{Signal} \\
\hline
প্রকৃতি & কাঁচা উপাদান & অর্থপূর্ণ ফলাফল & data বহনের রূপ \\
\hline
ব্যবহার & processing ও storage & decision making & transmission \\
\hline
উদাহরণ & 85, Rifat, image file & GPA result, report & electric pulse, light wave \\
\hline
\end{tabular}
\end{center}

\begin{examplebox}{সহজ উদাহরণ}
পরীক্ষায় একজন শিক্ষার্থীর subject-wise নম্বরগুলো হলো data। সেই নম্বর থেকে GPA বের করলে সেটি information। আর এই result online server থেকে mobile-এ পাঠানোর সময় electrical বা electromagnetic signal ব্যবহৃত হয়।
\end{examplebox}

\subsection{Analog Signal ও Digital Signal}

\begin{definitionbox}{Analog signal}
যে signal-এর মান সময়ের সাথে continuousভাবে পরিবর্তিত হয়, তাকে analog signal বলা হয়।
\end{definitionbox}

Analog signal সাধারণত smooth wave আকারে থাকে। মানুষের কণ্ঠস্বর, পুরনো telephone signal এবং radio wave analog signal-এর উদাহরণ হতে পারে। Analog signal continuous হওয়ায় noise বা disturbance দ্বারা বেশি প্রভাবিত হতে পারে।

\begin{definitionbox}{Digital signal}
যে signal-এর মান discrete বা নির্দিষ্ট স্তরে পরিবর্তিত হয় এবং সাধারণত 0 ও 1 দিয়ে প্রকাশ করা হয়, তাকে digital signal বলা হয়।
\end{definitionbox}

Digital signal computer system-এর জন্য বেশি উপযোগী, কারণ computer binary system অর্থাৎ 0 ও 1 ব্যবহার করে কাজ করে। Digital signal noise-এর বিরুদ্ধে তুলনামূলক বেশি reliable এবং error detection/correction সহজ।

Digital signal noise-এর বিরুদ্ধে reliable হলেও এটি সবসময় error-free নয়। বাস্তবে error detection, correction, encoding এবং protocol দরকার হয়।
\end{mistakebox}

\subsection{কমিউনিকেশন সিস্টেমের সংক্ষিপ্ত ইতিহাস}

Communication system ধীরে ধীরে উন্নত হয়েছে। প্রথমে মানুষ চিঠি, সংকেত ও telegraph ব্যবহার করত। পরে telephone, radio, television, computer network, internet, Wi-Fi, mobile communication এবং cloud computing communication ব্যবস্থাকে দ্রুত ও বিশ্বব্যাপী করেছে।

\begin{center}
\begin{tabular}{|p{0.18\textwidth}|p{0.30\textwidth}|p{0.36\textwidth}|}
\hline
\textbf{সময়/ধাপ} & \textbf{ঘটনা/প্রযুক্তি} & \textbf{গুরুত্ব} \\
\hline
Telegraph যুগ & Electric signal দিয়ে বার্তা পাঠানো & দূরবর্তী communication দ্রুত হয়। \\
\hline
Telephone যুগ & Voice communication & সরাসরি কথা বলা সম্ভব হয়। \\
\hline
Radio/TV যুগ & Wireless broadcasting & এক sender থেকে অনেক receiver-এ বার্তা যায়। \\
\hline
Computer network & Device-to-device data sharing & LAN, WAN ও internet-এর ভিত্তি তৈরি হয়। \\
\hline
Mobile network & Cellular communication & চলমান অবস্থায় voice ও data service সম্ভব হয়। \\
\hline
Cloud যুগ & Internet-based storage/service & যেকোনো স্থান থেকে resource ব্যবহার সম্ভব হয়। \\
\hline
\end{tabular}
\end{center}

\begin{keypointbox}{গুরুত্বপূর্ণ অবদানকারী}
\begin{itemize}
\item \textbf{Claude Shannon}: Information theory ও communication theory-এর ভিত্তি তৈরিতে গুরুত্বপূর্ণ অবদান রাখেন।
\item \textbf{Alexander Graham Bell}: Telephone উদ্ভাবনের সাথে যুক্ত অন্যতম গুরুত্বপূর্ণ ব্যক্তি।
\item \textbf{Vint Cerf ও Robert Kahn}: TCP/IP ধারণা ও Internet architecture উন্নয়নে গুরুত্বপূর্ণ অবদান রাখেন।
\item \textbf{Robert Metcalfe}: Ethernet প্রযুক্তির উন্নয়নে গুরুত্বপূর্ণ অবদান রাখেন।
\end{itemize}
\end{keypointbox}

\subsection{ব্যান্ডউইডথ}

\begin{definitionbox}{Bandwidth}
কোনো communication channel নির্দিষ্ট সময়ে কত পরিমাণ data বহন করতে পারে, সেই ক্ষমতাকে bandwidth বলা হয়।
\end{definitionbox}

Digital communication-এ bandwidth সাধারণত bits per second বা bps দিয়ে প্রকাশ করা হয়। Bandwidth বেশি হলে একই সময়ে বেশি data পাঠানো সম্ভব হয়। তবে bandwidth বেশি হলেই সবসময় user experience ভালো হবে—এমন নয়; latency, server capacity, network congestion, signal strength এবং device performance-ও speed-এর উপর প্রভাব ফেলে।

\subsection{Bandwidth, Data Rate ও Latency}

Bandwidth, data rate এবং latency কাছাকাছি concept হলেও এক নয়।

\begin{center}
\begin{tabular}{|p{0.22\textwidth}|p{0.58\textwidth}|}
\hline
\textbf{Term} & \textbf{ব্যাখ্যা} \\
\hline
Bandwidth & communication channel-এর সর্বোচ্চ data বহন ক্ষমতা। \\
\hline
Data rate & বাস্তবে প্রতি সেকেন্ডে কত data transfer হচ্ছে। \\
\hline
Latency & data sender থেকে receiver-এ পৌঁছাতে যে বিলম্ব হয়। \\
\hline
Throughput & বাস্তবে সফলভাবে transfer হওয়া data-এর পরিমাণ। \\
\hline
\end{tabular}
\end{center}

\begin{examplebox}{সহজ তুলনা}
রাস্তার lane বেশি হলে একসাথে বেশি গাড়ি চলতে পারে—এটি bandwidth-এর মতো। কিন্তু গাড়ির গতি, traffic jam এবং signal delay বাস্তব speed কমাতে পারে—এগুলো data rate ও latency-এর সাথে তুলনীয়।
\end{examplebox}

\begin{mistakebox}{Common mistake}
অনেকে bandwidth ও internet speed একই মনে করে। Bandwidth হলো capacity, আর actual speed নির্ভর করে bandwidth, latency, congestion, server response এবং device performance-এর উপর।
\end{mistakebox}

\end{boardbox}

\subsection{Bandwidth Calculation}

Data communication-এ bandwidth calculation খুব গুরুত্বপূর্ণ। সাধারণভাবে,

\[
\text{Bandwidth} = \frac{\text{Total transmitted data}}{\text{Time}}
\]

\begin{examplebox}{সমাধান ১}
একটি channel দিয়ে ৪ সেকেন্ডে ৮০০০ bit data পাঠানো হলো। Bandwidth কত?

\[
\text{Bandwidth} = \frac{8000}{4} = 2000 \text{ bps}
\]

অতএব, bandwidth = 2000 bps।
\end{examplebox}

\begin{examplebox}{সমাধান ২}
একটি file 20,000 bit এবং এটি 10 second-এ transfer হলো।

\[
\text{Bandwidth} = \frac{20000}{10} = 2000 \text{ bps}
\]

অতএব, channel-এর data transfer rate = 2000 bps।
\end{examplebox}

\begin{examplebox}{সমাধান ৩: Byte থেকে bit}
একটি file-এর size 500 Byte এবং এটি 2 second-এ transfer হলো। Bandwidth কত?

প্রথমে Byte থেকে bit-এ রূপান্তর করতে হবে।

\[
500 \text{ Byte} = 500 \times 8 = 4000 \text{ bit}
\]

\[
\text{Bandwidth} = \frac{4000}{2} = 2000 \text{ bps}
\]

অতএব, bandwidth = 2000 bps।
\end{examplebox}

\begin{mistakebox}{Common mistake}
Bandwidth calculation-এ অনেকেই Byte ও bit গুলিয়ে ফেলে। মনে রাখতে হবে: 1 Byte = 8 bits। প্রশ্নে যদি Byte দেওয়া থাকে, প্রথমে bit-এ convert করতে হবে।
\end{mistakebox}

\subsection{Exam-style প্রশ্ন ও উত্তর}

\begin{boardbox}{সংক্ষিপ্ত প্রশ্ন: ডেটা কমিউনিকেশন কী?}
ডিজিটাল বা অ্যানালগ data-কে নির্দিষ্ট communication medium ও protocol ব্যবহার করে এক device থেকে অন্য device-এ প্রেরণ ও গ্রহণ করার প্রক্রিয়াকে data communication বলা হয়। এতে sender, receiver, message, medium এবং protocol থাকে। যেমন—email পাঠানো, mobile message, online class ইত্যাদি।
\end{boardbox}

\begin{boardbox}{সংক্ষিপ্ত প্রশ্ন: Bandwidth কেন গুরুত্বপূর্ণ?}
Bandwidth গুরুত্বপূর্ণ কারণ এটি নির্ধারণ করে কোনো communication channel নির্দিষ্ট সময়ে কত data বহন করতে পারবে। Bandwidth বেশি হলে audio, video, file transfer এবং internet browsing দ্রুত ও কার্যকর হয়। তবে বাস্তব speed latency, network congestion ও device performance-এর উপরও নির্ভর করে।
\end{boardbox}

\begin{boardbox}{পার্থক্য: Analog signal ও Digital signal}
Analog signal continuousভাবে পরিবর্তিত হয়, কিন্তু digital signal discrete value ধারণ করে। Analog signal noise-এ বেশি প্রভাবিত হয়, আর digital signal তুলনামূলকভাবে reliable। পুরনো radio system analog signal ব্যবহার করে, কিন্তু computer network ও internet communication digital signal ব্যবহার করে।
\end{boardbox}
