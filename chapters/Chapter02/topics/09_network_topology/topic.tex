\subsection{ভূমিকা}

বর্তমান যুগে computer network শিক্ষা প্রতিষ্ঠান, অফিস, ব্যাংক, ব্যবসা প্রতিষ্ঠান, গবেষণাগার এবং ঘরোয়া ব্যবহারে অত্যন্ত গুরুত্বপূর্ণ। একাধিক computer বা device পরস্পরের সাথে যুক্ত হলে data আদান-প্রদান, hardware ও software resource sharing, দ্রুত যোগাযোগ এবং centralized management সহজ হয়।

তবে network তৈরি করার সময় শুধু device-গুলোকে cable বা wireless link দিয়ে যুক্ত করলেই যথেষ্ট নয়। কোন computer কোনটির সাথে যুক্ত থাকবে, data কোন পথে চলাচল করবে, কোন device কেন্দ্রীয় ভূমিকা পালন করবে এবং কোনো অংশ নষ্ট হলে network-এর উপর কী প্রভাব পড়বে - এসব বিষয় পরিকল্পনা করে নির্ধারণ করতে হয়। এই পরিকল্পিত সংযোগ বিন্যাসই network topology।

একই সংখ্যক device নিয়েও ভিন্ন ভিন্ন topology তৈরি করা যায়। topology বদলালে cable খরচ, data transmission-এর গতি, reliability, security, troubleshooting এবং maintenance-এর ধরনও বদলে যায়। তাই network তৈরি করার আগে cost, speed, reliability, security এবং future expansion বিবেচনা করে উপযুক্ত topology নির্বাচন করা প্রয়োজন।

\begin{definitionbox}{Network topology}
কম্পিউটার network-এ ব্যবহৃত computer, server, switch, router, hub এবং সংযোগ মাধ্যমগুলোর বিন্যাস বা সংযোগ কাঠামো, যার মাধ্যমে data আদান-প্রদানের পথ নির্ধারিত হয়, তাকে network topology বলে।
\end{definitionbox}

\subsection{টপোলজির প্রয়োজনীয়তা}

Network topology network-এর গঠন, খরচ, গতি, নিরাপত্তা ও নির্ভরযোগ্যতার উপর সরাসরি প্রভাব ফেলে। সঠিক topology নির্বাচন করলে network পরিচালনা সহজ হয়, fault দ্রুত ধরা যায় এবং ভবিষ্যতে network বড় করা সুবিধাজনক হয়।

\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item \textbf{Network planning:} কোন device কোথায় বসবে এবং কীভাবে সংযুক্ত হবে তা আগে থেকে বোঝা যায়।
\item \textbf{Data path:} data কোন node বা device হয়ে destination-এ যাবে তা নির্ধারণ করা যায়।
\item \textbf{Cost control:} cable, switch, router, port ও installation খরচ অনুমান করা যায়।
\item \textbf{Fault detection:} কোনো cable, node বা central device নষ্ট হলে কোন অংশ প্রভাবিত হবে তা দ্রুত বোঝা যায়।
\item \textbf{Performance:} data collision, delay ও traffic load কমানোর জন্য উপযুক্ত design নেওয়া যায়।
\item \textbf{Expansion:} ভবিষ্যতে নতুন computer, branch বা department যোগ করার পরিকল্পনা সহজ হয়।
\end{itemize}

\subsection{Physical ও Logical topology}

\textbf{Physical topology} হলো network device ও cable-এর বাস্তব বা দৃশ্যমান সংযোগ বিন্যাস। অর্থাৎ computer, switch, router, cable ইত্যাদি বাস্তবে কোথায় বসানো হয়েছে এবং কীভাবে সংযুক্ত হয়েছে, সেটি physical topology।

\textbf{Logical topology} হলো network-এর ভিতরে data কীভাবে চলাচল করে তার যৌক্তিক পথ। device-গুলো বাস্তবে যেভাবেই সংযুক্ত থাকুক, data কোন নিয়মে এবং কোন পথে এক node থেকে অন্য node-এ যায়, সেটিই logical topology।

\begin{examplebox}{উদাহরণ}
একটি lab-এ সব computer যদি একটি switch-এর সাথে cable দিয়ে যুক্ত থাকে, তাহলে physical structureটি Star topology। কিন্তু data flow switch configuration ও communication protocol অনুযায়ী logicalভাবে নিয়ন্ত্রিত হয়।
\end{examplebox}

\subsection{নেটওয়ার্ক টপোলজির গুরুত্বপূর্ণ উপাদান}

\begin{keypointbox}{গুরুত্বপূর্ণ শব্দ}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item \textbf{Node:} network-এ যুক্ত প্রতিটি computer, printer, server, phone, router বা অন্য device।
\item \textbf{Link:} দুইটি node-এর মধ্যে সংযোগ। এটি wired cable বা wireless connection হতে পারে।
\item \textbf{Hub:} central device, যা data পেলে সাধারণত connected সব device-এর দিকে পাঠায়।
\item \textbf{Switch:} hub-এর তুলনায় উন্নত device; destination address দেখে নির্দিষ্ট device-এ data পাঠায়।
\item \textbf{Router:} এক network থেকে অন্য network-এ data পাঠায়; internet connection ও বড় network-এ গুরুত্বপূর্ণ।
\item \textbf{Backbone:} network-এর প্রধান সংযোগ line বা main cable, যার সাথে একাধিক device বা branch যুক্ত থাকে।
\end{itemize}
\end{keypointbox}

\subsection{Star Topology}

\end{definitionbox}

\topicheading{কার্যপ্রণালি}
Star topology-তে কোনো computer data পাঠালে data প্রথমে central hub/switch-এর কাছে যায়। এরপর central device data-টি destination computer-এ পাঠায়। ফলে hub/switch এই network-এর communication centre হিসেবে কাজ করে।

\topicheading{বৈশিষ্ট্য}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item সব node একটি central device-এর সাথে যুক্ত থাকে।
\item central device হিসেবে সাধারণত hub বা switch ব্যবহৃত হয়।
\item প্রতিটি node-এর জন্য আলাদা cable লাগে।
\item একটি node নষ্ট হলে সাধারণত অন্য node-এর communication বন্ধ হয় না।
\item central device নষ্ট হলে পুরো network অচল হতে পারে।
\end{itemize}

\begin{keypointbox}{Hub ও Switch ব্যবহারের পার্থক্য}
Hub data পেলে সাধারণত সব connected device-এর দিকে পাঠায়। Switch destination address দেখে নির্দিষ্ট device-এ data পাঠায়। তাই modern Star topology-তে switch ব্যবহার করলে performance ও security তুলনামূলক ভালো হয়।
\end{keypointbox}

\topicheading{সুবিধা}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item সহজে স্থাপন করা যায়।
\item ত্রুটি নির্ণয় সহজ।
\item একটি computer নষ্ট হলেও পুরো network বন্ধ হয় না।
\item নতুন device যোগ করা সহজ।
\item school lab, office LAN ও home network-এ বেশি ব্যবহৃত হয়।
\end{itemize}

\topicheading{অসুবিধা}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item hub/switch নষ্ট হলে পুরো network বন্ধ হয়ে যেতে পারে।
\item প্রতিটি node-এর জন্য আলাদা cable লাগায় খরচ বাড়ে।
\item অনেক device একসাথে traffic করলে central device-এ চাপ পড়ে।
\end{itemize}

\begin{examplebox}{ব্যবহার}
School বা college computer lab, office LAN, small business network এবং home router-based network-এ Star topology বেশি ব্যবহৃত হয়।
\end{examplebox}

\begin{boardbox}{পরীক্ষার সংক্ষিপ্ত উত্তর}
Star topology হলো এমন network topology যেখানে সব computer বা node একটি কেন্দ্রীয় hub বা switch-এর সাথে সংযুক্ত থাকে এবং data আদান-প্রদান সেই কেন্দ্রীয় device-এর মাধ্যমে সম্পন্ন হয়।
\end{boardbox}

\begin{boardbox}{সৃজনশীল উত্তরের মূল পয়েন্ট}
Star topology-তে central hub/switch-এর ভূমিকা, একটি computer নষ্ট হলেও network সচল থাকা, নতুন node যোগের সুবিধা এবং central device নষ্ট হলে পুরো network বন্ধ হওয়ার ঝুঁকি উল্লেখ করলে উত্তর পূর্ণাঙ্গ হয়।
\end{boardbox}

\subsection{Ring Topology}

\begin{definitionbox}{Ring Network Topology}
Ring topology হলো এমন একটি network topology যেখানে প্রতিটি computer তার পূর্ববর্তী ও পরবর্তী computer-এর সাথে যুক্ত হয়ে একটি বৃত্তাকার communication path তৈরি করে।

\topicheading{কার্যপ্রণালি}
Ring topology-তে data সাধারণত একটি নির্দিষ্ট direction-এ এক node থেকে পরবর্তী node-এ যায়। গন্তব্য node না আসা পর্যন্ত data ধারাবাহিকভাবে node অতিক্রম করে।

\topicheading{বৈশিষ্ট্য}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item প্রতিটি node তার পাশের দুইটি node-এর সাথে যুক্ত থাকে।
\item network structure closed loop বা বৃত্তাকার হয়।
\item সাধারণত central hub/switch থাকে না।
\item প্রতিটি node data গ্রহণ করে এবং প্রয়োজন হলে পরবর্তী node-এ পাঠায়।
\item open end না থাকায় Bus topology-এর মতো terminator দরকার হয় না।
\end{itemize}

\topicheading{সুবিধা}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item data চলাচলের নির্দিষ্ট path থাকে।
\item collision তুলনামূলক কম হতে পারে।
\item প্রতিটি node signal regenerate করতে পারে।
\item ছোট বা controlled network-এ ব্যবহার করা যায়।
\end{itemize}

\topicheading{অসুবিধা}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item একটি node বা cable নষ্ট হলে পুরো ring network প্রভাবিত হতে পারে।
\item নতুন node যোগ করতে network সাময়িকভাবে বন্ধ করতে হতে পারে।
\item fault বের করা star topology-এর তুলনায় কঠিন।
\item আধুনিক LAN-এ এটি খুব বেশি ব্যবহৃত হয় না।
\end{itemize}

\begin{examplebox}{ব্যবহার}
পুরনো Token Ring network, নির্দিষ্ট control system বা loop-based closed network-এ Ring topology দেখা যেতে পারে।
\end{examplebox}

\begin{boardbox}{পরীক্ষার সংক্ষিপ্ত উত্তর}
Ring topology হলো এমন network topology যেখানে প্রতিটি computer তার পাশের দুইটি computer-এর সাথে যুক্ত হয়ে একটি বৃত্তাকার network তৈরি করে।
\end{boardbox}

\begin{boardbox}{সৃজনশীল উত্তরের মূল পয়েন্ট}
Ring topology ব্যাখ্যার সময় closed loop, central device না থাকা, node-to-node data flow, collision কম হওয়া এবং একটি node/link নষ্ট হলে network ক্ষতিগ্রস্ত হওয়ার দিকগুলো লিখতে হবে।
\end{boardbox}

\topicheading{বৈশিষ্ট্য}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]

\item data একই communication line share করে।
\item central hub বা switch থাকে না।
\item ছোট network হলে setup সহজ ও খরচ কম।
\item backbone cable network-এর সবচেয়ে গুরুত্বপূর্ণ অংশ।
\end{itemize}

\begin{keypointbox}{Terminator}
Bus topology-তে backbone cable-এর দুই প্রান্তে terminator ব্যবহার করা হয়। Terminator signal reflection বা bounce back কমিয়ে data transmission স্থিতিশীল রাখতে সাহায্য করে।
\end{keypointbox}

\topicheading{সুবিধা}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item cable কম লাগে।
\item ছোট network তৈরি করতে খরচ কম।
\item setup তুলনামূলক সহজ।
\item temporary বা demonstration network-এর জন্য উপযোগী।
\end{itemize}

\topicheading{অসুবিধা}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item backbone cable নষ্ট হলে পুরো network বন্ধ হয়ে যেতে পারে।
\item computer সংখ্যা বেশি হলে network ধীর হয়ে যায়।
\item data collision বেশি হতে পারে।
\item fault শনাক্ত করা কঠিন।
\item বড় network-এর জন্য উপযোগী নয়।
\end{itemize}

\begin{examplebox}{ব্যবহার}
ছোট বা temporary network এবং পুরনো Ethernet network-এ Bus topology ব্যবহার দেখা যেত। আধুনিক office LAN-এ সাধারণত Star topology বেশি ব্যবহৃত হয়।
\end{examplebox}

\begin{boardbox}{পরীক্ষার সংক্ষিপ্ত উত্তর}
Bus topology হলো এমন network topology যেখানে সব computer বা node একটি প্রধান cable বা backbone cable-এর সাথে যুক্ত থাকে।
\end{boardbox}

\begin{boardbox}{সৃজনশীল উত্তরের মূল পয়েন্ট}
Bus topology-এর উত্তরে backbone cable, কম cable খরচ, ছোট network-এ উপযোগিতা, data collision এবং backbone নষ্ট হলে পুরো network বন্ধ হওয়ার বিষয়গুলো লিখলে উত্তর শক্তিশালী হয়।
\end{boardbox}

Tree topology হলো এমন একটি network topology যেখানে network-এর computer বা device গুলো গাছের শাখা-প্রশাখার মতো স্তরভিত্তিক বা hierarchical বিন্যাসে যুক্ত থাকে।

\topicheading{কার্যপ্রণালি}
Tree topology-তে network স্তরভিত্তিকভাবে কাজ করে। উপরের স্তরের device নিচের স্তরের device গুলোকে connect বা control করে। কোনো computer data পাঠালে data প্রথমে local switch-এ যায়, তারপর প্রয়োজন হলে upper-level switch বা root device হয়ে destination branch-এ পৌঁছে।

\topicheading{বৈশিষ্ট্য}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item এটি hierarchical বা স্তরভিত্তিক topology।
\item root/core device থেকে branch ও sub-branch তৈরি হয়।
\item একাধিক star network root বা backbone-এর মাধ্যমে যুক্ত থাকতে পারে।
\item বড় network-কে ছোট অংশে ভাগ করে পরিচালনা করা যায়।
\item root বা parent device গুরুত্বপূর্ণ ভূমিকা পালন করে।
\end{itemize}

\begin{keypointbox}{Tree topology কেন Star + Bus-এর মতো}
Tree topology-তে নিচের branch-গুলো অনেক সময় Star topology-এর মতো হয়, আর branch-গুলো একটি root/backbone-এর মাধ্যমে যুক্ত থাকে। তাই বড় network গঠনে এটি Star ও Bus topology-এর বৈশিষ্ট্য মিলিয়ে ব্যবহার করতে পারে।
\end{keypointbox}

\topicheading{সুবিধা}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item বড় network তৈরি ও পরিচালনা করা যায়।
\item department-wise বা floor-wise network design করা সহজ।
\item network expand করা তুলনামূলক সহজ।
\item fault নির্দিষ্ট branch-এ সীমাবদ্ধ থাকতে পারে।
\item school, college, office বা campus network-এর জন্য উপযোগী।
\end{itemize}

\topicheading{অসুবিধা}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item root বা backbone device নষ্ট হলে network-এর বড় অংশ অচল হতে পারে।
\item cable বেশি লাগে।
\item configuration তুলনামূলক জটিল।
\item maintenance খরচ বেশি হতে পারে।
\end{itemize}

\begin{examplebox}{ব্যবহার}
বড় office, শিক্ষা প্রতিষ্ঠান, campus network, bank বা department-wise network design-এ Tree topology উপযোগী।
\end{examplebox}

\begin{boardbox}{পরীক্ষার সংক্ষিপ্ত উত্তর}
Tree topology হলো এমন network topology যেখানে computer বা device গুলো গাছের শাখা-প্রশাখার মতো স্তরভিত্তিকভাবে বিন্যস্ত থাকে।
\end{boardbox}

\begin{boardbox}{সৃজনশীল উত্তরের মূল পয়েন্ট}
Tree topology-এর উত্তরে hierarchical structure, root/core switch, department-wise branch, সহজ expansion এবং root/backbone নষ্ট হলে বড় অংশ অচল হওয়ার ঝুঁকি উল্লেখ করতে হবে।
\end{boardbox}

\subsection{Mesh Topology}

\begin{definitionbox}{Mesh Network Topology}
Mesh topology হলো এমন একটি network topology যেখানে network-এর প্রতিটি node একাধিক node-এর সাথে, এবং full mesh ক্ষেত্রে প্রতিটি node অন্য সব node-এর সাথে সরাসরি সংযুক্ত থাকে।
\end{definitionbox}

\topicheading{কার্যপ্রণালি}
একটি node থেকে data সরাসরি বা বিকল্প path দিয়ে গন্তব্য node-এ যেতে পারে। কোনো link নষ্ট হলেও data অন্য link দিয়ে যেতে পারে। তাই Mesh topology reliability ও fault tolerance-এর জন্য শক্তিশালী।

\topicheading{Mesh topology-এর প্রকারভেদ}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item \textbf{Full Mesh:} প্রতিটি node network-এর অন্য সব node-এর সাথে সরাসরি যুক্ত থাকে।
\item \textbf{Partial Mesh:} প্রতিটি node সব node-এর সাথে যুক্ত থাকে না; কিছু গুরুত্বপূর্ণ node multiple links রাখে।
\end{itemize}

\begin{boardbox}{Full Mesh link formula}
Full mesh topology-তে যদি node সংখ্যা \(n\) হয়, তবে মোট direct link সংখ্যা হবে:
\[
\frac{n(n-1)}{2}
\]
যেমন, 4টি node থাকলে link সংখ্যা \( \frac{4 \times 3}{2}=6 \) টি। 10টি computer হলে link সংখ্যা \( \frac{10 \times 9}{2}=45 \) টি।
\end{boardbox}

\topicheading{বৈশিষ্ট্য}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item প্রতিটি node একাধিক node-এর সাথে connected থাকে।
\item full mesh হলে প্রতিটি node অন্য সব node-এর সাথে সরাসরি যুক্ত থাকে।
\item data চলাচলের জন্য একাধিক path থাকে।
\item reliability ও fault tolerance বেশি।
\item cable, port, installation ও configuration বেশি লাগে।
\end{itemize}

\topicheading{সুবিধা}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item reliability সবচেয়ে বেশি।
\item একটি link নষ্ট হলেও data অন্য path দিয়ে যেতে পারে।
\item data transmission নিরাপদ ও দ্রুত হতে পারে।
\item fault সহজে isolate করা যায়।
\item high-security বা critical network-এর জন্য উপযোগী।
\end{itemize}

\topicheading{অসুবিধা}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item cable অনেক বেশি লাগে।
\item installation খরচ বেশি।
\item configuration ও maintenance জটিল।
\item প্রতিটি device-এ বেশি port দরকার হয়।
\item বড় network-এ full mesh সবসময় practical নয়।
\end{itemize}

\begin{examplebox}{ব্যবহার}
Bank network, military communication, data center, wireless mesh network এবং critical communication system-এ Mesh topology ধারণা ব্যবহৃত হয়।
\end{examplebox}

\begin{boardbox}{পরীক্ষার সংক্ষিপ্ত উত্তর}
Mesh topology হলো এমন network topology যেখানে প্রতিটি node একাধিক বা সব node-এর সাথে সরাসরি সংযুক্ত থাকে, ফলে data চলাচলের জন্য একাধিক পথ থাকে।
\end{boardbox}

\begin{boardbox}{সৃজনশীল উত্তরের মূল পয়েন্ট}
Mesh topology বিশ্লেষণে multiple path, full/partial mesh, link formula, reliability, fault tolerance, security এবং বেশি cable/port খরচের বিষয়গুলো রাখতে হবে।
\end{boardbox}

Hybrid topology-তে network-এর প্রয়োজন অনুযায়ী বিভিন্ন topology একসাথে ব্যবহার করা হয়। যেমন, কোনো প্রতিষ্ঠানের প্রতিটি department Star topology ব্যবহার করতে পারে, আবার department switch-গুলো backbone link দিয়ে যুক্ত হতে পারে।

\topicheading{Hybrid topology-এর common form}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item \textbf{Star-Bus Hybrid:} একাধিক Star network একটি backbone cable-এর সাথে যুক্ত থাকে।
\item \textbf{Star-Ring Hybrid:} কয়েকটি Star network ring আকারে যুক্ত হতে পারে।
\item \textbf{Star-Mesh Hybrid:} user computer Star topology-তে এবং গুরুত্বপূর্ণ server/router Mesh topology-তে যুক্ত থাকে।
\end{itemize}

\topicheading{বৈশিষ্ট্য}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item দুই বা ততোধিক topology-এর সমন্বয়ে তৈরি হয়।
\item বড় network design করার জন্য উপযোগী।
\item organization-এর প্রয়োজন অনুযায়ী flexible structure তৈরি করা যায়।
\item network expansion সহজ হতে পারে।
\item design, installation ও maintenance তুলনামূলক জটিল।
\end{itemize}

\topicheading{সুবিধা}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item flexible network design করা যায়।
\item বড় network তৈরি করা যায়।
\item প্রয়োজন অনুযায়ী topology নির্বাচন করা যায়।
\item কোনো অংশে সমস্যা হলে পুরো network সবসময় বন্ধ হয় না।
\item performance ও scalability ভালো হতে পারে।
\end{itemize}

\topicheading{অসুবিধা}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]

\item installation ও maintenance খরচ বেশি।

\begin{keypointbox}{Board ও admission quick facts}
\begin{itemize}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=0pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item central hub/switch/router থাকলে topology সাধারণত Star।
\item backbone cable শব্দ থাকলে Bus topology মনে রাখতে হবে।
\item circular বা closed loop থাকলে Ring topology।
\item reliability, redundant path, multiple direct link বা \( \frac{n(n-1)}{2} \) link formula থাকলে Mesh topology।
\item গাছের শাখা-প্রশাখার মতো স্তরভিত্তিক network হলে Tree topology।
\item দুই বা ততোধিক topology মিলে network হলে Hybrid topology।
\end{itemize}
\end{keypointbox}

\begin{mistakebox}{সাধারণ ভুল}
Star topology মানেই Internet topology নয়। Star হলো local arrangement, যেখানে central device থাকে। আবার Wi-Fi network অনেক সময় physical দিক থেকে Star-এর মতো হলেও logical data flow device ও access point-এর configuration অনুযায়ী ভিন্ন হতে পারে।
\end{mistakebox}

\textbf{সৃজনশীল অনুশীলন}
\begin{enumerate}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=1pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item একটি কলেজের ICT lab-এ ২৫টি computer আছে। সব computer একটি switch-এর সাথে যুক্ত। একটি computer নষ্ট হলেও অন্য computer গুলো কাজ করতে পারে। কিন্তু switch নষ্ট হলে পুরো lab network বন্ধ হয়ে যায়। উদ্দীপকের network কোন topology? কারণ, সুবিধা ও দুর্বলতা ব্যাখ্যা করো।
\item একটি ছোট অফিসে ৮টি computer একটি মাত্র main cable-এর সাথে যুক্ত। main cable নষ্ট হলে সব computer-এর communication বন্ধ হয়ে যায়। অফিসটি কম খরচে network তৈরি করেছে। উদ্দীপকের network কোন topology? ছোট network-এ এটি উপযোগী হলেও বড় network-এ কেন উপযোগী নয়, বিশ্লেষণ করো।
\item একটি ব্যাংকের server network-এ প্রতিটি server অন্য server-এর সাথে সরাসরি যুক্ত। একটি link নষ্ট হলেও data অন্য link দিয়ে গন্তব্যে পৌঁছাতে পারে। তবে network তৈরি করতে অনেক cable ও port দরকার হয়েছে। উদ্দীপকের topology শনাক্ত করে reliability ও ব্যয়বহুল হওয়ার কারণ লেখ।
\item একটি university campus network-এ প্রতিটি department-এর computer গুলো Star topology-তে যুক্ত। আবার department switch গুলো একটি main backbone-এর সাথে যুক্ত। Server room-এ গুরুত্বপূর্ণ server গুলো Mesh topology-তে যুক্ত করা হয়েছে। উদ্দীপকের network Hybrid topology কেন, সুবিধা ও maintenance জটিলতা ব্যাখ্যা করো।
\end{enumerate}

\textbf{সৃজনশীল উত্তরের ইঙ্গিত}
\begin{enumerate}[leftmargin=1.0cm,labelsep=0.35cm,itemsep=1pt,topsep=2pt,parsep=0pt,partopsep=0pt]
\item উদ্দীপকে সব computer central switch-এর সাথে যুক্ত, তাই এটি Star topology। সুবিধা হিসেবে একটি computer নষ্ট হলেও অন্যগুলো সচল থাকা এবং fault isolation সহজ হওয়া লিখবে। দুর্বলতা হিসেবে switch নষ্ট হলে পুরো lab network বন্ধ হওয়ার কথা লিখবে।
\item উদ্দীপকে সব computer একটি main cable বা backbone-এর সাথে যুক্ত, তাই এটি Bus topology। কম cable ও কম খরচের কারণে ছোট network-এ উপযোগী; কিন্তু backbone নষ্ট হলে পুরো network বন্ধ এবং traffic বেশি হলে collision/speed problem হয়।
\item প্রতিটি server অন্য server-এর সাথে সরাসরি যুক্ত এবং link নষ্ট হলেও বিকল্প path আছে, তাই এটি Mesh topology। Reliability বেশি কারণ multiple path থাকে; ব্যয়বহুল কারণ cable, port, connector ও configuration বেশি লাগে।
\item উদ্দীপকে department-এ Star, department switch-গুলোর মধ্যে backbone এবং server room-এ Mesh connection আছে; তাই এটি Hybrid topology। বড় campus network-এ flexible design ও expansion সুবিধা দিলেও maintenance, configuration ও troubleshooting জটিল।
\end{enumerate}
\end{practicebox}

\begin{recapbox}
বর্তমান যুগে computer network বিভিন্ন প্রতিষ্ঠান, শিক্ষা প্রতিষ্ঠান, অফিস, ব্যাংক ও গবেষণাগারে অত্যন্ত গুরুত্বপূর্ণ। একটি network কত দ্রুত, নিরাপদ ও নির্ভরযোগ্যভাবে কাজ করবে তা অনেকাংশে topology নির্বাচনের উপর নির্ভর করে। তাই HSC ICT শিক্ষার্থীদের জন্য network topology জানা গুরুত্বপূর্ণ, কারণ এটি computer networking অধ্যায়ের একটি মৌলিক বিষয়।
\end{recapbox}
