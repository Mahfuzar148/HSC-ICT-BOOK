\section{ক্লাউড কম্পিউটিং}

\begin{learningbox}
এই টপিক শেষে শিক্ষার্থী পারবে:
\begin{itemize}
\item ক্লাউড কম্পিউটিং কী তা ব্যাখ্যা করতে।
\item ক্লাউড কম্পিউটিং কীভাবে কাজ করে তা বুঝতে।
\item Cloud storage, cloud application, cloud server ও data center-এর ধারণা নিতে।
\item IaaS, PaaS ও SaaS service model-এর পার্থক্য লিখতে।
\item Public, Private, Hybrid ও Community cloud deployment model ব্যাখ্যা করতে।
\item ক্লাউড কম্পিউটিং-এর সুবিধা, অসুবিধা ও নিরাপত্তা ঝুঁকি বিশ্লেষণ করতে।
\item বাস্তব উদাহরণ থেকে cloud computing service শনাক্ত করতে।
\item বোর্ড পরীক্ষার জন্য short answer, comparison answer ও CQ answer লিখতে।
\end{itemize}
\end{learningbox}

\subsection{বাস্তব জীবনের শুরু}

ধরো, তুমি Google Drive-এ একটি file upload করলে। এরপর mobile, laptop বা অন্য কোনো computer থেকে internet ব্যবহার করে সেই file access করতে পারো। File-টি তোমার নিজের computer-এর hard disk-এ না থেকে কোনো cloud provider-এর server বা data center-এ সংরক্ষিত থাকে।

আবার Gmail, Google Docs, Microsoft OneDrive, Dropbox, YouTube, Facebook, online class platform, online exam system—এসব service ব্যবহার করার সময় আমরা সরাসরি cloud computing-এর সুবিধা নিচ্ছি।

অর্থাৎ cloud computing হলো এমন ব্যবস্থা যেখানে storage, software, server, database, networking বা computing power নিজের local computer-এ না রেখে internet-এর মাধ্যমে remote server থেকে ব্যবহার করা যায়।

\begin{keypointbox}{মূল ধারণা}
Cloud computing বুঝতে তিনটি প্রশ্ন মনে রাখো:
\begin{itemize}
\item data বা software কোথায় আছে? Local computer-এ, নাকি remote server-এ?
\item user কীভাবে access করছে? Internet-এর মাধ্যমে।
\item user কি নিজের hardware কিনছে, নাকি service হিসেবে ব্যবহার করছে?
\end{itemize}
\end{keypointbox}

\subsection{ক্লাউড কম্পিউটিং}

\begin{definitionbox}{ক্লাউড কম্পিউটিং}
Internet-এর মাধ্যমে remote server, storage, database, networking, software, platform ও computing resource প্রয়োজন অনুযায়ী ব্যবহার করার প্রযুক্তিকে cloud computing বলা হয়।
\end{definitionbox}

সহজভাবে, cloud computing হলো \textbf{internet-based computing service}। এখানে user নিজে server, storage বা software infrastructure তৈরি না করেও cloud provider-এর resource ব্যবহার করতে পারে।

\begin{examplebox}{সহজ উদাহরণ}
\begin{itemize}
\item Google Drive-এ file রাখা হলো cloud storage।
\item Gmail ব্যবহার করা হলো cloud-based email service।
\item Google Docs ব্যবহার করে online document edit করা হলো cloud application।
\item কোনো website AWS, Azure বা Google Cloud server-এ host করা হলো cloud hosting।
\end{itemize}
\end{examplebox}

\subsection{ক্লাউড কম্পিউটিং-এর মূল উপাদান}

Cloud computing system সাধারণত কয়েকটি গুরুত্বপূর্ণ component নিয়ে গঠিত।

\begin{keypointbox}{Cloud computing components}
\begin{itemize}
\item \textbf{Client device}: user-এর mobile, laptop, desktop বা tablet।
\item \textbf{Internet}: user ও cloud server-এর মধ্যে connection path।
\item \textbf{Cloud server}: remote server যেখানে application বা service চলে।
\item \textbf{Cloud storage}: data, file, image, video বা backup রাখার জায়গা।
\item \textbf{Data center}: অনেক server, storage ও networking device নিয়ে তৈরি বড় infrastructure।
\item \textbf{Virtualization}: physical hardware-কে virtual resource হিসেবে ভাগ করে ব্যবহার করার প্রযুক্তি।
\item \textbf{Cloud application}: browser বা app দিয়ে ব্যবহার করা cloud-based software।
\end{itemize}
\end{keypointbox}

\subsection{Cloud Computing কীভাবে কাজ করে?}

Cloud computing-এ user কোনো service ব্যবহার করতে চাইলে request internet-এর মাধ্যমে cloud provider-এর server-এ যায়। Server request process করে data বা result user device-এ পাঠিয়ে দেয়।

\begin{keypointbox}{কাজের ধাপ}
\begin{enumerate}
\item user mobile/computer থেকে internet ব্যবহার করে cloud service access করে।
\item request cloud provider-এর server বা data center-এ যায়।
\item cloud server required software, database বা storage ব্যবহার করে request process করে।
\item processed result বা file user device-এ ফিরে আসে।
\item user local device-এ সব resource install না করেও service ব্যবহার করতে পারে।
\end{enumerate}
\end{keypointbox}

\subsection{Cloud Computing-এর Essential Characteristics}

Standard cloud computing বোঝার জন্য পাঁচটি essential characteristic খুব গুরুত্বপূর্ণ।

\subsubsection{On-demand Self-service}

User প্রয়োজন অনুযায়ী storage, processing power বা software service নিজে request করে ব্যবহার করতে পারে। প্রতিবার service provider-এর সাথে manual যোগাযোগ করার প্রয়োজন হয় না।

\subsubsection{Broad Network Access}

Cloud service internet বা network-এর মাধ্যমে mobile, laptop, desktop, tablet ইত্যাদি বিভিন্ন device থেকে access করা যায়।

\subsubsection{Resource Pooling}

Cloud provider অনেক user-এর জন্য server, storage, database, network ইত্যাদি resource pool করে রাখে। User প্রয়োজন অনুযায়ী সেই shared resource ব্যবহার করে।

\subsubsection{Rapid Elasticity}

User-এর চাহিদা বাড়লে resource দ্রুত বাড়ানো যায় এবং চাহিদা কমলে resource কমানো যায়। যেমন website-এ visitor বেশি হলে server capacity বাড়ানো।

\subsubsection{Measured Service}

Cloud resource ব্যবহার পরিমাপ করা যায়। User কত storage, bandwidth বা computing power ব্যবহার করল তা monitor করা যায় এবং অনেক ক্ষেত্রে usage অনুযায়ী bill হয়।

\subsection{IaaS}

\begin{definitionbox}{IaaS}
IaaS বা Infrastructure as a Service হলো cloud service model যেখানে user virtual server, storage, network এবং computing infrastructure internet-এর মাধ্যমে service হিসেবে ব্যবহার করে।
\end{definitionbox}

IaaS-এ provider hardware infrastructure দেয়, আর user operating system, application, database ইত্যাদি নিজের প্রয়োজন অনুযায়ী configure করতে পারে।

\begin{keypointbox}{IaaS-এর বৈশিষ্ট্য}
\begin{itemize}
\item virtual machine, server, storage ও network resource পাওয়া যায়।
\item physical server কিনতে হয় না।
\item user-এর control তুলনামূলক বেশি।
\item website hosting, backup server, testing server ইত্যাদিতে ব্যবহার করা যায়।
\item technical knowledge বেশি প্রয়োজন হতে পারে।
\end{itemize}
\end{keypointbox}

\begin{examplebox}{উদাহরণ}
AWS EC2, Microsoft Azure Virtual Machines, Google Compute Engine—এসব IaaS ধারণার উদাহরণ হিসেবে বোঝানো যায়।
\end{examplebox}

\subsection{PaaS}

\begin{definitionbox}{PaaS}
PaaS বা Platform as a Service হলো cloud service model যেখানে application তৈরি, test, deploy ও run করার জন্য required platform, runtime, database ও development tools service হিসেবে দেওয়া হয়।
\end{definitionbox}

PaaS-এ developer server manage করার ঝামেলা ছাড়াই application develop করতে পারে। Provider infrastructure ও platform manage করে।

\begin{keypointbox}{PaaS-এর বৈশিষ্ট্য}
\begin{itemize}
\item application development-এর platform দেয়।
\item server ও operating system management provider করে।
\item developer coding, testing ও deployment-এ focus করতে পারে।
\item web application ও mobile application development-এ উপযোগী।
\item user-এর control IaaS-এর তুলনায় কম, কিন্তু ease of use বেশি।
\end{itemize}
\end{keypointbox}

\begin{examplebox}{উদাহরণ}
Google App Engine, Microsoft Azure App Service, Heroku—এসব PaaS ধারণার উদাহরণ হিসেবে ব্যাখ্যা করা যায়।
\end{examplebox}

\subsection{SaaS}

\begin{definitionbox}{SaaS}
SaaS বা Software as a Service হলো cloud service model যেখানে software internet-এর মাধ্যমে ready-to-use service হিসেবে ব্যবহার করা যায়।
\end{definitionbox}

SaaS-এ user সাধারণত browser বা mobile app দিয়ে software ব্যবহার করে। Software install, update, maintenance—এসব কাজ provider manage করে।

\begin{keypointbox}{SaaS-এর বৈশিষ্ট্য}
\begin{itemize}
\item ready-made software service পাওয়া যায়।
\item local computer-এ software install না করেও ব্যবহার করা যায়।
\item update ও maintenance provider করে।
\item সাধারণ user-এর জন্য সবচেয়ে সহজ cloud service model।
\item internet connection প্রয়োজন।
\end{itemize}
\end{keypointbox}

\begin{examplebox}{উদাহরণ}
Gmail, Google Docs, Microsoft 365, Dropbox, Canva, Zoom—এসব SaaS-এর উদাহরণ হিসেবে ব্যবহার করা যায়।
\end{examplebox}

\subsection{Public Cloud}

\begin{definitionbox}{Public Cloud}
যে cloud infrastructure কোনো third-party cloud provider পরিচালনা করে এবং internet-এর মাধ্যমে বহু user বা organization service হিসেবে ব্যবহার করে, তাকে public cloud বলা হয়।
\end{definitionbox}

Public cloud সাধারণত cost-effective এবং scalable। তবে sensitive data-এর ক্ষেত্রে security ও compliance carefully manage করতে হয়।

\begin{keypointbox}{Public cloud-এর বৈশিষ্ট্য}
\begin{itemize}
\item third-party provider দ্বারা operated হয়।
\item internet-এর মাধ্যমে access করা যায়।
\item অনেক user shared infrastructure ব্যবহার করে।
\item cost তুলনামূলক কম হতে পারে।
\item scalability বেশি।
\item user-এর control private cloud-এর তুলনায় কম।
\end{itemize}
\end{keypointbox}

\subsection{Private Cloud}

\begin{definitionbox}{Private Cloud}
যে cloud infrastructure একটি নির্দিষ্ট organization বা প্রতিষ্ঠানের জন্য dedicated থাকে এবং সেই organization-ই মূলত ব্যবহার করে, তাকে private cloud বলা হয়।
\end{definitionbox}

Private cloud-এ control ও security বেশি হতে পারে। ব্যাংক, বড় corporate office, government বা sensitive data-based organization private cloud ব্যবহার করতে পারে।

\begin{keypointbox}{Private cloud-এর বৈশিষ্ট্য}
\begin{itemize}
\item একটি নির্দিষ্ট organization-এর জন্য dedicated।
\item control ও customization বেশি।
\item sensitive data management-এ উপযোগী।
\item cost public cloud-এর তুলনায় বেশি হতে পারে।
\item skilled maintenance team দরকার হতে পারে।
\end{itemize}
\end{keypointbox}

\subsection{Hybrid Cloud}

\begin{definitionbox}{Hybrid Cloud}
Public cloud ও private cloud একত্রে ব্যবহার করে যে cloud environment তৈরি হয়, তাকে hybrid cloud বলা হয়।
\end{definitionbox}

Hybrid cloud-এ organization sensitive data private cloud-এ রাখতে পারে এবং high scalability বা public service-এর জন্য public cloud ব্যবহার করতে পারে।

\begin{keypointbox}{Hybrid cloud-এর বৈশিষ্ট্য}
\begin{itemize}
\item public ও private cloud-এর combination।
\item flexibility বেশি।
\item sensitive workload private cloud-এ রাখা যায়।
\item scalable service public cloud-এ রাখা যায়।
\item design ও management তুলনামূলক complex হতে পারে।
\end{itemize}
\end{keypointbox}

\subsection{Community Cloud}

\begin{definitionbox}{Community Cloud}
যে cloud infrastructure একই ধরনের উদ্দেশ্য, policy বা security requirement থাকা একাধিক organization একসাথে ব্যবহার করে, তাকে community cloud বলা হয়।
\end{definitionbox}

Community cloud সাধারণত education, healthcare, government বা research-based organizations-এর মধ্যে shared requirement থাকলে ব্যবহার করা যেতে পারে।

\begin{keypointbox}{Community cloud-এর বৈশিষ্ট্য}
\begin{itemize}
\item common interest থাকা organizations একসাথে ব্যবহার করে।
\item cost share করা যায়।
\item common policy ও security requirement follow করা যায়।
\item public cloud-এর চেয়ে controlled হতে পারে।
\item management agreement দরকার হয়।
\end{itemize}
\end{keypointbox}

\subsection{Cloud Storage}

\begin{definitionbox}{Cloud Storage}
Internet-এর মাধ্যমে remote server বা cloud data center-এ data, file, image, video বা backup সংরক্ষণ করার ব্যবস্থাকে cloud storage বলা হয়।
\end{definitionbox}

Cloud storage-এ file upload করলে user internet connection থাকলে যেকোনো জায়গা থেকে file access করতে পারে। File share, sync ও backup করার জন্য cloud storage খুব useful।

\begin{keypointbox}{Cloud storage-এর বৈশিষ্ট্য}
\begin{itemize}

\item যেকোনো device থেকে data access করা যায়।
\item file sharing ও synchronization সহজ।
\item automatic backup রাখা যায়।
\item internet না থাকলে access problem হতে পারে।
\item privacy ও security setting ঠিকভাবে manage করতে হয়।
\end{itemize}
\end{keypointbox}

\subsection{Cloud Computing-এর সুবিধা}

Cloud computing modern ICT system-এ খুব গুরুত্বপূর্ণ, কারণ এটি কম খরচে দ্রুত, scalable এবং flexible computing service ব্যবহার করতে সাহায্য করে।

\begin{keypointbox}{Cloud computing-এর সুবিধা}
\begin{itemize}
\item \textbf{Cost saving}: server, storage ও hardware কিনতে কম খরচ হতে পারে।
\item \textbf{Scalability}: প্রয়োজন অনুযায়ী resource বাড়ানো বা কমানো যায়।
\item \textbf{Accessibility}: internet থাকলে যেকোনো জায়গা থেকে service access করা যায়।
\item \textbf{Backup}: data backup রাখা সহজ।
\item \textbf{Collaboration}: একই file বা document অনেক user একসাথে কাজ করতে পারে।
\item \textbf{Automatic update}: software update provider manage করতে পারে।
\item \textbf{Disaster recovery}: data center-based backup থাকলে data recovery সহজ হতে পারে।
\item \textbf{Flexibility}: user device বা location independent service ব্যবহার করতে পারে।
\end{itemize}

\subsection{Cloud Computing-এর অসুবিধা ও সীমাবদ্ধতা}

Cloud computing সুবিধাজনক হলেও কিছু limitation ও risk আছে।

\begin{keypointbox}{Cloud computing-এর অসুবিধা}
\begin{itemize}
\item internet connection ছাড়া service ব্যবহার করা কঠিন হতে পারে।
\item data privacy ও security risk থাকতে পারে।
\item provider-এর server down হলে service disruption হতে পারে।
\item long-term usage cost ঠিকভাবে monitor না করলে খরচ বাড়তে পারে।
\item data অন্য provider-এ স্থানান্তর করা কঠিন হতে পারে।
\item sensitive data cloud-এ রাখলে legal/compliance issue থাকতে পারে।
\item user-এর full control সব ক্ষেত্রে থাকে না।
\end{itemize}
\end{keypointbox}

\subsection{Cloud Security}

Cloud computing-এ data remote server-এ থাকে। তাই security খুব গুরুত্বপূর্ণ। Password, encryption, access control, backup, multi-factor authentication এবং proper sharing permission cloud security নিশ্চিত করতে সাহায্য করে।

\begin{keypointbox}{Cloud security practice}
\begin{itemize}
\item strong password ব্যবহার করা।
\item multi-factor authentication চালু রাখা।
\item file sharing permission carefully manage করা।
\item sensitive data encrypt করা।
\item regular backup রাখা।
\item trusted cloud provider ব্যবহার করা।
\item public Wi-Fi থেকে sensitive cloud account login এড়িয়ে চলা।
\end{itemize}
\end{keypointbox}

\subsection{Common Mistake Zone}

\begin{mistakebox}{Common mistake ১}
অনেকে cloud computing মানেই শুধু online storage মনে করে। আসলে cloud computing-এর মধ্যে storage ছাড়াও server, database, software, platform, networking ও computing power অন্তর্ভুক্ত।
\end{mistakebox}

\begin{mistakebox}{Common mistake ২}
Google Drive cloud storage, কিন্তু Gmail/Google Docs cloud application বা SaaS service হিসেবেও বোঝানো যায়।
\end{mistakebox}

\begin{mistakebox}{Common mistake ৩}
Public cloud মানে data public হয়ে যায়—এটি ভুল। Public cloud shared infrastructure হলেও user data logical security ও access control দ্বারা আলাদা রাখা হয়।
\end{mistakebox}

\begin{mistakebox}{Common mistake ৪}
Cloud computing ব্যবহার করলে security দরকার নেই—এটি ভুল। Strong password, MFA, access control ও encryption জরুরি।
\end{mistakebox}

\begin{mistakebox}{Common mistake ৫}
IaaS, PaaS ও SaaS একই নয়। IaaS infrastructure দেয়, PaaS development platform দেয়, SaaS ready software দেয়।
\end{mistakebox}

\subsection{Exam-style প্রশ্ন ও উত্তর}

\begin{boardbox}{সংক্ষিপ্ত প্রশ্ন: Cloud computing কী?}
Internet-এর মাধ্যমে remote server, storage, database, networking, software, platform ও computing resource প্রয়োজন অনুযায়ী ব্যবহার করার প্রযুক্তিকে cloud computing বলা হয়। এতে user নিজের local computer-এ সব resource না রেখেও online service ব্যবহার করতে পারে। যেমন—Google Drive, Gmail, Google Docs, Dropbox ইত্যাদি।
\end{boardbox}

\begin{boardbox}{সংক্ষিপ্ত প্রশ্ন: Cloud storage কী?}
Internet-এর মাধ্যমে remote server বা cloud data center-এ data, file, image, video বা backup সংরক্ষণ করার ব্যবস্থাকে cloud storage বলা হয়। যেমন—Google Drive, OneDrive, Dropbox, iCloud।
\end{boardbox}

\begin{boardbox}{সংক্ষিপ্ত প্রশ্ন: IaaS কী?}
IaaS বা Infrastructure as a Service হলো cloud service model যেখানে user virtual server, storage, network ও computing infrastructure internet-এর মাধ্যমে service হিসেবে ব্যবহার করে। এতে physical server কিনতে হয় না।
\end{boardbox}

\begin{boardbox}{সংক্ষিপ্ত প্রশ্ন: PaaS কী?}
PaaS বা Platform as a Service হলো cloud service model যেখানে application তৈরি, test, deploy ও run করার জন্য development platform, runtime ও tools service হিসেবে দেওয়া হয়। এটি developer-দের জন্য উপযোগী।
\end{boardbox}

\begin{boardbox}{সংক্ষিপ্ত প্রশ্ন: SaaS কী?}
SaaS বা Software as a Service হলো cloud service model যেখানে software internet-এর মাধ্যমে ready-to-use service হিসেবে ব্যবহার করা যায়। user সাধারণত browser বা app দিয়ে software ব্যবহার করে। যেমন—Gmail, Google Docs, Microsoft 365।
\end{boardbox}

\begin{boardbox}{সংক্ষিপ্ত প্রশ্ন: Public cloud কী?}
যে cloud infrastructure কোনো third-party cloud provider পরিচালনা করে এবং internet-এর মাধ্যমে বহু user বা organization service হিসেবে ব্যবহার করে, তাকে public cloud বলা হয়। এটি scalable ও cost-effective হতে পারে।
\end{boardbox}

\begin{boardbox}{সংক্ষিপ্ত প্রশ্ন: Private cloud কী?}
যে cloud infrastructure একটি নির্দিষ্ট organization বা প্রতিষ্ঠানের জন্য dedicated থাকে এবং সেই organization-ই মূলত ব্যবহার করে, তাকে private cloud বলা হয়। sensitive data ও বেশি control দরকার হলে private cloud ব্যবহার করা যায়।
\end{boardbox}

\begin{boardbox}{সংক্ষিপ্ত প্রশ্ন: Hybrid cloud কী?}
Public cloud ও private cloud একত্রে ব্যবহার করে যে cloud environment তৈরি হয়, তাকে hybrid cloud বলা হয়। এতে sensitive data private cloud-এ রাখা যায় এবং scalable service public cloud-এ চালানো যায়।
\end{boardbox}

\begin{boardbox}{পার্থক্য: IaaS, PaaS ও SaaS}
IaaS infrastructure যেমন virtual server, storage ও network service হিসেবে দেয়। PaaS application development ও deployment-এর platform দেয়। SaaS ready-made software internet-এর মাধ্যমে ব্যবহার করতে দেয়। IaaS-এ user control বেশি, PaaS-এ development সহজ, আর SaaS সাধারণ user-এর জন্য সবচেয়ে সহজ।
\end{boardbox}

\begin{boardbox}{পার্থক্য: Public cloud ও Private cloud}
Public cloud third-party provider পরিচালনা করে এবং বহু user internet-এর মাধ্যমে shared infrastructure ব্যবহার করে। Private cloud নির্দিষ্ট একটি organization-এর জন্য dedicated থাকে, তাই control ও customization বেশি হয়। Public cloud cost-effective ও scalable, কিন্তু private cloud sensitive data management-এর জন্য বেশি উপযোগী হতে পারে।
\end{boardbox}

\subsection{১ মিনিটে রিভিশন}

\begin{recapbox}
\begin{itemize}
\item Cloud computing হলো internet-এর মাধ্যমে remote computing resource ব্যবহার।
